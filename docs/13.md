# CI, сборка пакетов и распространение

Как footstrap собирается в пакеты и раздаётся. Тема **noarch** (`LUCI_PKGARCH:=all`)
и поддерживает **OpenWrt 24.10** (`.ipk`/opkg) и **25.12+** (`.apk`/apk).

## GitHub Actions (`.github/workflows/build.yml`)

Триггеры: push в `main`, тег `v*`, любой `pull_request` или ручной `workflow_dispatch`.
Джобы: `check` → `lint` → `build` (`needs: [check, lint]`) → `release` (только на тег).

`permissions: contents: read` на уровне workflow; write объявляет только job `release`.
Раньше write-скоуп был workflow-wide, то есть выдавался каждому `pull_request`-прогону —
включая `npm ci` в lint, а значит lifecycle-скриптам всех dev-зависимостей.

### `check` — гейты без node (7 шагов)

Нужны только `sh`, `awk`, `python3`, `perl` и (для шаблонов) `cmake` — стоят секунды и не
могут сломать buildbot OpenWrt, где node нет и не будет.

1. **`sh -n`** по всем скриптам пакета, `install.sh`, `tools/*.sh` и `uci-defaults`.
   `tools/` в глобе потому, что `release-notes.sh` запускается **только** job'ом `release` —
   синтаксическая ошибка в нём обнаруживалась в момент, когда это дороже всего.
2. **Компиляция ucode-шаблонов.** Собирает `ucode` из коммита, запиненного в
   `luci-upstream.pin`, и гоняет `ucode -T -c` (это LuCI'шный trycompile) по каждому `.ut`.
   Импорты `luci.core`/`uci` — рантайм роутера, на раннере их нет, поэтому они **застаблены**
   (`-L`): шаблон реально парсится, тела стабов не исполняются (`-c` компилирует, не запускает).
   Гейт появился потому, что у шаблонов **не было парсера вообще**: luci.mk копирует `ucode/`
   verbatim, лишняя скобка в `header.ut` собиралась зелёной, релизилась — и у всех LuCI молча
   откатывался на другую тему (docs/01).
3. **ACL — валидный JSON.** rpcd **пропускает** нечитаемый файл в `acl.d` и молчит: лишняя
   запятая = `file.exec` на self-update не выдан никому, бейдж обновления и кнопка «Обновить»
   умирают с «Access denied» у всех, и больше ничто этого не замечает.
4. **`build-css.sh /tmp/cascade.css`** — заодно **бюджет размера**: скрипт падает, если
   минифицированный `cascade.css` больше `FS_CSS_BUDGET` (по умолчанию 117 760 Б ≈ 115 КБ;
   фактически сегодня **~112 КБ**, бюджет 117 760 Б). uhttpd отдаёт `/www/luci-static/*.css` **без сжатия** —
   это байты на проводе.
5. **Бюджет шрифтов**: суммарно `fonts/*.woff2` ≤ 71 680 Б (фактически 68 488). Не потолок, а
   **храповик** — было 102 400, пока шипался жирный моно-фейс.
6. **`audit.py --strict`** — undefined `var()`, затенённые декларации, чтение export-тира из
   `styles/`, мёртвые base-декларации, лишние `!important`, цветовые литералы.
7. **i18n**: `update-po.sh --check` — падает, если `.pot` устарел (строку добавили, скрипт не
   перезапустили) или хоть один `msgstr` пуст. Строка в `_()` без перевода рисуется
   по-английски **молча** — так весь поповер Appearance и оставался английским на русском LuCI.

### `lint` — npm-гейты (11 проверок)

Живут **только** в CI: у buildbot'а нет node, и luci.mk он не нужен — тот копирует `htdocs/`
и `ucode/` как есть. Ничего из `package.json` **не шипается**. Локально всё сразу — `npm run
check`.

| Шаг | Что ловит |
|---|---|
| `eslint` (`htdocs`) | в т.ч. `wrap-regex` — запрещает `return /re/…`, форму, на которой jsmin ломается |
| `stylelint` (`styles/**`) | только correctness + инварианты проекта (не форматтер) |
| `a11y-gallery.mjs` | axe-core, WCAG 2.2 AA по `docs/gallery.html`, матрица {light,dark} × {footstrap,hicontrast} |
| `export-tier.mjs` | контракт `--*-color-*` с чужими `luci-app-*` — axe его не видит: их виджетов в галерее нет |
| `css-metrics.mjs` | храповик: `!important` ≤ 33, специфичность, пустые правила |
| `fs-orphans.mjs` | мёртвые `fs-*`-селекторы (безопасно только в нашем неймспейсе) |
| `css-dup.mjs` | одинаковые тела деклараций под разными гардами — их не поймает ни один линтер |
| `mirror.mjs` | `@mirror`-запиненные копии (CSS **и** shell) побайтово совпадают |
| `axes.mjs` | пре-пейнт в `head.ut` согласован с живыми аппликаторами Appearance |
| `jsmin-verify.mjs` | **единственная** проверка, ловящая тихую порчу от jsmin (exit 0) |
| бюджет JS | размер **после** минификации: ≤ 47 104 Б (фактически 46 249) |

Про `jsmin-verify`: luci.mk минифицирует наш JS jsmin'ом на buildbot'е. jsmin решает,
`/` — регекс или деление, по **одному** предыдущему символу, и ни `n` (из `return`), ни `>`
(из `=>`) в allow-list не входят — `return /re/` заставляет его сожрать остаток файла **и
выйти с кодом 0** (openwrt/luci#8299, #8020, #8021, #8256). **Нулевой exit не доказывает
ничего.** Поэтому CI собирает тот же самый jsmin из `luci-base/src/jsmin.c` и доказывает, что
поток токенов вывода **идентичен** исходному (acorn). Список шипаемых JS не выписан руками —
`find htdocs -name '*.js'`: luci.mk копирует `htdocs/` целиком, значит это и есть шипаемое
множество.

### `luci-upstream.pin` — заимствованные тулы

Единственный источник запиненного коммита `openwrt/luci` **и sha256** двух инструментов,
которые CI **скачивает и ИСПОЛНЯЕТ**:

- `jsmin.c` — компилируется и решает, безопасен ли наш JS;
- `build/i18n-scan.pl` — запускается перлом (`update-po.sh`) и решает, полон ли каталог.

Взятые с движущегося `master`, эти гейты были бы «чем угодно, что упстрим запушил последним»;
sha256 говорит это вслух. Там же — `UCODE_PIN` (по коммиту, без sha256: имя git-объекта **и
есть** контрольная сумма его содержимого, а генерируемые GitHub'ом архивы побайтово
нестабильны). Коммит раньше был выписан дважды — в workflow и в `update-po.sh`, каждый с
комментарием «bump together» и ничем не подкреплённый.

### Матрица форматов (`build`)

| fmt | channel | SDK | CONFIG_USE_APK |
|---|---|---|---|
| apk | snapshot | `snapshots/targets/x86/64` (apk-тулчейн 25.12+) | `y` |
| ipk | 24.10 | последний `24.10.x` release SDK | `n` |

Шаги каждой сборки:

1. Резолв версии: из тега (`v0.3.6` → `0.3.6`) или fallback `0.<date>.<run>`.
2. Скачать SDK — URL резолвится **динамически** (grep по dir-листингу, не хардкод
   toolchain-версии; поддержаны `.tar.zst` и `.tar.xz`), затем **сверить sha256** с
   опубликованным OpenWrt `sha256sums`. SDK — наименее проверенный вход в этом репозитории и
   единственный, что попадает в пакет, который пользователи ставят: два **линтера** пинятся по
   коммиту и хешу, а тулчейн, собирающий артефакт, приезжал на доверии.
3. `rsync` каталога `luci-theme-footstrap/` в `package/luci-theme-footstrap/` SDK (сохраняя
   симлинки), `feeds update -a && install -a`.
4. `.config`: `CONFIG_PACKAGE_luci-theme-footstrap=m` (+ `CONFIG_USE_APK=y` для apk),
   `make defconfig`, `make package/…/compile V=s FOOTSTRAP_VERSION=<ver>`.
5. Ассерт, что в build-дереве есть `footstrap-theme.*.lmo`: каталог компилируется **внутрь**
   пакета (`Build/Prepare` → `po2lmo`), в git его нет, а молча пропавший `.lmo` = каждая `_()`
   рисует английский msgid и никто ни о чём не сообщает.
6. Собрать `.apk`/`.ipk` из `bin/` в `dist/` и проверить, что там **ровно один** пакет на
   формат. Это не гигиена, а защита старых роутеров: `footstrap-selfupdate.sh`, который у людей
   **уже стоит**, выбирает ассет как `grep -E '\.apk$' | head -1`, а GitHub отдаёт ассеты
   **отсортированными по имени**. В 0.8.4 в релизе появился второй пакет
   (`luci-i18n-footstrap-ru`) — он сортируется раньше `luci-theme-footstrap-*`, и кнопка
   «Обновить» у всех ставила 6-КБ каталог вместо темы, рапортуя об успехе (issue #6). Скрипт на
   роутере удалённо не починить — неизменным должен остаться **релиз**. Если второй пакет
   когда-нибудь действительно понадобится, он обязан сортироваться **после** темы в **обеих**
   схемах (apk `name-1.2.3-r1.apk`, ipk `name_1.2.3-r1_all.ipk` — заметьте, `-` < `_`).
7. Job `release` (только на тег): скачивает артефакты обоих форматов, генерирует тело релиза
   из CHANGELOG (`tools/release-notes.sh` — жирный лид каждого пункта секции `## [x.y.z]`),
   `softprops/action-gh-release` цепляет оба пакета.

Новый релиз: см. правила CHANGELOG в CLAUDE.md (переименовать `[Unreleased]` → коммит → тег
**этого** коммита), затем `git tag vX.Y.Z && git push origin vX.Y.Z` (пушить **на оба**
remote'а).

## Решения в Makefile (почему так)

Полный разбор — **docs/02**. Здесь только то, что касается сборки:

- **`LUCI_MINIFY_CSS:=0`** — критично. luci.mk гонит CSS через **csstidy**, а тот достаточно
  стар, чтобы манглить `:has()`, `color-mix()`, вложенный `calc()`: пакет ставится, верстка
  разваливается. Минифицирует вместо него `build-css.sh` (~112 КБ, бюджет 117 760 Б).
- **`LUCI_MINIFY_JS` НЕ выставлен** — дефолт `1`, то есть jsmin, **и тема этого хочет**:
  комментарии — 57 % исходника JS, jsmin даёт 127 КБ → 47 КБ, а uhttpd отдаёт `/www` без
  сжатия. Выставить `0` = утроить шипаемый JS. Безопасность обеспечивают `wrap-regex` и
  `jsmin-verify` (см. выше), а не отключение минификатора.
- **`$(TOPDIR)/feeds/luci/luci.mk`** (абсолютный) — относительный `../../luci.mk` работает
  только когда пакет лежит в `feeds/luci/themes/`; CI rsync'ает его в `package/<name>/`.
- **`LUCI_PKGARCH:=all`** — темы архитектурно-независимы; один пакет на все таргеты.

## Установка и цепочка доверия

Скрипт [`install.sh`](../install.sh) определяет `apk`/`opkg` и ставит нужный формат из
последнего (или заданного) релиза:

```sh
wget -qO- https://raw.githubusercontent.com/VizzleTF/luci-theme-footstrap/main/install.sh | sh
# закрепить версию:  ... | sh -s v0.3.6
```

Вручную — **raw-файл из релиза** (НЕ zip-артефакт из Actions):

```sh
apk add --allow-untrusted luci-theme-footstrap-*.apk   # 25.12+
opkg install luci-theme-footstrap_*.ipk                # 24.10
```

**`--allow-untrusted` = подписи пакета НЕТ.** И `install.sh` (его льют `curl | sh` **под
root**), и `footstrap-selfupdate.sh` (бэкенд кнопки «Обновить», за ACL rpcd) ставят пакет
именно так — значит вся цепочка доверия состоит ровно из двух звеньев, и оба несущие:

1. **Проверенный TLS-канал.** Никогда `curl -k`, никогда `--no-check-certificate` — и **не в
   качестве ретрая** после неудачной проверенной попытки: неудача **и есть** случай MITM, а
   `ca-bundle` входит в `DEFAULT_PACKAGES` OpenWrt, так что небезопасный путь не покупает
   ничего. Схема редиректа тоже пинится (`--proto-redir '=https'`): ассет релиза прыгает на
   `objects.githubusercontent.com`. Хост ассета — по allow-list'у (`github.com`,
   `objects.githubusercontent.com`, `release-assets.githubusercontent.com`).
2. **sha256, который GitHub публикует для ассета** (`@.assets[*].digest` в API релиза).
   Несовпадение = отказ ставить; **отсутствие digest'а — тоже отказ**, а не предупреждение.
   Хеш едет тем же каналом, что и URL, поэтому он не защита от скомпрометированного
   `api.github.com`; он защита от подменённой или обрезанной загрузки с **CDN ассетов** — а
   это другой хост.

**`curl` НЕ входит в дефолтный набор пакетов OpenWrt** (базовый образ несёт `uclient-fetch`).
Self-updater поэтому откатывается на `uclient-fetch`, а не тянет curl зависимостью — благодаря
чему рантайм-список остаётся `+luci-base`. Раньше он жёстко требовал curl, и на стоковом
роутере кнопка «Обновить» умирала с «cannot reach the GitHub release API». `jsonfilter` и
`sha256sum` в базовом образе **есть**.

Два скрипта **не могут разделить файл** (installer — это `curl | sh`, он бежит **до** того,
как появится пакет, в котором лежала бы библиотека), поэтому их `fetch()`, allow-list хостов и
разбор ассетов запинены `@mirror gh/{fetch,asset-host,asset-urls}` и держатся побайтово
одинаковыми `tools/mirror.mjs`. Это не церемония: три копии **уже** успели разойтись (разный
порядок бэкендов, у одной первый инструмент без таймаута, у одной не было пина схемы редиректа).

Перевод едет внутри пакета темы (`/usr/lib/lua/luci/i18n/footstrap-theme.<lang>.lmo`) —
см. docs/02.

## Формат пакетов + типичная ошибка

- **apk** (25.12+) — формат Alpine apk-tools (OpenWrt перешёл на него в 25.12).
- **ipk** (24.10) — gzip-tar: `./debian-binary` (2.0) + `./control.tar.gz` + `./data.tar.gz`,
  ustar-заголовки. Идентичен формату родных OpenWrt-ipk.

**«Malformed package file» в opkg** почти всегда = ставят НЕ тот файл: zip-обёртку
GitHub-артефакта или `.apk` вместо `.ipk`.

Проверить `.ipk` настоящим opkg можно **вручную** (в CI такого прогона нет — сборка ipk
проверяется только тем, что SDK 24.10 её произвёл):

```sh
docker run --rm -v /path/pkg.ipk:/tmp/p.ipk openwrt/rootfs:x86-64-24.10.4 \
  sh -c 'mkdir -p /var/lock; opkg install --nodeps /tmp/p.ipk'
```
