# Подводные камни 25.12+ и чек-лист новой темы

## Камни совместимости

1. **Только ucode.** Темы со старыми `luasrc/view/themes/*.htm` (Lua-шаблоны) на
   25.12 без пакета `luci-lua-runtime` не работают. Многие темы с гитхаба
   (старые форки argon/material) до сих пор несут `.htm` — не копировать оттуда
   слепо. Ориентир — только официальные темы из openwrt/luci master.

2. **apk (25.12+) и opkg (24.10).** 25.12 перешёл на apk: пакеты `.apk`,
   `apk list --installed` (не `opkg list-installed`). Но тема noarch собирается и
   под 24.10 (`.ipk`/opkg) — CI даёт оба формата (docs/13). Версии пакетов должны
   соответствовать правилам apk (строже opkg); `luci.mk` генерирует сам — не
   задавать PKG_VERSION вручную с экзотикой (в CI footstrap подставляет версию из
   тега через `FOOTSTRAP_VERSION`, в SDK-сборке без `.git` иначе версии нет —
   docs/13). uci-defaults срабатывают штатно.

3. **Меню рендерится клиентом.** Нельзя сгенерить меню сервером в header.ut —
   дерево приходит асинхронно (`ui.menu.load()`). Контейнеры до загрузки пустые:
   тема обязана нормально выглядеть в момент "меню ещё нет" (bootstrap прячет их
   `style="display:none"`).

4. **`data-page` на body обязателен.** CSS luci-base и приложений (statistics,
   firewall) таргетирует `[data-page="admin-..."]`. Убрали атрибут — сломали
   чужие стили.

5. **`node.css` и `css` в header.** Отдельные страницы (напр. статистика) просят
   дополнительный CSS. Не выводите — страницы виснут визуально. Аналогично
   `{{ resource }}/cbi.js` и скрипт переводов — обязательные включения.

6. **blank_page.** Забыли обработать — на странице логина появится меню/шапка
   с битыми ссылками (юзер не аутентифицирован).

7. **Не хардкодить пути.** Всегда `{{ media }}` и `{{ resource }}`, не
   `/luci-static/mytheme` строкой: `media` = `basename(mediaurlbase)`, и тема вполне
   может оказаться смонтирована под другим basename (легаси-запись в `luci.themes`,
   ручная правка `mediaurlbase`). Хардкод — это 404 на CSS и шрифтах.

8. **postrm обязателен.** Без него после `apk del` остаётся запись в
   `luci.themes`, и если тема была активной — у пользователя fallback-ошибка
   при каждом заходе, пока не переключит тему руками. `postrm` footstrap ещё и
   переводит `luci.main.mediaurlbase` на `bootstrap` (если тот стоит), чтобы не
   полагаться на рантайм-fallback, и сносит маркер первой установки.

9. **Права юзера ≠ root.** LuCI поддерживает не-root логины (rpcd ACL). Меню
   приходит уже отфильтрованным — menu-JS не должен предполагать наличие узлов
   (`node?.children` с проверками, как в оригинале).

10. **`meta viewport` + мобильный CSS.** Забыть viewport — мобильная вёрстка
    мертва. Bootstrap выносит мобильное в отдельный `mobile.css` с media query
    на `<link>` — грузится всегда, применяется по ширине.

11. **initramfs/recovery и «нет пароля»** — предупреждения рисует header ТЕМЫ
    (не luci-base). Выкинете их — пользователь не узнает, что система в
    recovery-режиме. Скопировать блоки из bootstrap.

12. **darkreader-lock.** Без `<meta name="darkreader-lock">` расширение Dark
    Reader перекрасит вашу тёмную тему в кашу.

13. **Минификаторы luci.mk — это ДВА разных инструмента, и выключать надо только один.**
    Ни то, ни другое не проявляется в dev-цикле (dev-sync копирует файлы как есть) —
    только в собранном пакете. См. docs/13.

    - **CSS: `LUCI_MINIFY_CSS:=0` — обязательно.** Здесь минификатор — **csstidy**, он
      достаточно стар, чтобы манглить `:has()`, `color-mix()` и вложенный `calc()`: пакет
      ставится, но вёрстка «съезжает» (симптом: cascade.css в пакете сильно меньше
      исходного). Минифицирует вместо него `build-css.sh`.
    - **JS: `LUCI_MINIFY_JS` НЕ трогать — дефолт `1`, и тема его хочет.** Здесь
      минификатор — **jsmin** (`luci-base/src/jsmin.c`; `luci-base/host` и так build-dep,
      т.е. на бильдботе он есть). Комментарии — 57% JS темы, jsmin ужимает 127 KB до 47 KB,
      а uhttpd отдаёт `/www` **без сжатия**, так что это и байты по проводу, и байты во
      флеше. Выключить его — значит платить за комментарии на устройстве с 8–16 MB флеша.

    **Опасность jsmin — другая, и она хуже, чем «съехавшая вёрстка».** Он решает, что такое
    `/` — регэксп или деление, по **одному** предыдущему символу; ни `n` (последняя буква
    `return`), ни `>` (из `=>`) не входят в его allow-list. Поэтому

    ```js
    return /^https?:\/\//i.test(a);    // jsmin читает // регэкспа как комментарий,
                                       // проглатывает ОСТАТОК ФАЙЛА и выходит с кодом 0
    return (/^https?:\/\//i.test(a));  // `(` в allow-list — безопасно
    ```

    Это не теория: openwrt/luci#8299, #8020, #8021, #8256. **Нулевой exit-код jsmin ничего
    не доказывает** — порча молчаливая. Отсюда два гейта в CI: eslint'овый `wrap-regex`
    (запрещает саму опасную форму) и `tools/jsmin-verify.mjs` (прогоняет jsmin по каждому
    файлу и требует, чтобы поток токенов на выходе совпал с исходным — единственная проверка,
    ловящая порчу с exit 0).

14. **Неактивные вкладки-панели → фантомный скролл.** Вкладочные формы (Network →
    DNS/Interfaces/DHCP, Firewall, Flash…) рендерят КАЖДУЮ вкладку как панель
    `<… data-tab-title>` в DOM сразу; активную помечают `data-tab-active="true"`.
    Bootstrap-база гасит неактивные `[data-tab-title] { height:0; opacity:0;
    overflow:hidden }`. В footstrap этого мало: панель схлопывается в 0, но её
    (клиппнутый) контент всё равно раздувает `document.documentElement.scrollHeight`
    — страница листается на сотни px в пустоту ниже футера (DNS: +792px, Interfaces:
    +585px). Причём панель бывает БЕЗ класса `.cbi-section` — dnsmasq рендерит
    вкладки плоским `<div data-tab-title>`, поэтому узкий селектор
    `.cbi-section[data-tab-title]` их пропускал. Лечение (styles/theme/30-tables.css):
    `[data-tab-title]:not([data-tab-active="true"]) { display:none; padding:0;
    margin:0; border:0 }` — вынести неактивную панель из потока целиком. Безопасно:
    пункты меню вкладок несут `data-tab`, но НЕ `data-tab-title`, так что бар не
    задет; переключение работает (клик → LuCI ставит `data-tab-active="true"` →
    `:not()` раскрывает). Симптом ловится не скриншотом, а `scrollHeight − footer.bottom`
    на любой вкладочной странице (должно быть ~0).

15. **Релиз с НЕСКОЛЬКИМИ ассетами ломает кнопку Update на всех роутерах в поле.**
    Реально случилось (v0.8.4, issue #6). Каталог переводов лежал в `po/`, а из `po/*`
    luci.mk выводит `LUCI_LANGUAGES` и печёт отдельный пакет `luci-i18n-<name>-<lang>` на
    язык. В релизе стало несколько ассетов — а самообновлятор, который **уже стоит** у
    пользователя, выбирает ассет так: `grep '\.apk$' | head -1`. GitHub отдаёт ассеты
    отсортированными **по имени**, `luci-i18n-…` сортируется раньше `luci-theme-…` → Update
    ставил 6-KB каталог вместо темы, рапортовал успех и предлагал то же обновление вечно.
    **Установленный самообновлятор нельзя починить удалённо — чинить приходится РЕЛИЗ**, то
    есть он обязан остаться «выбираемым» тем скриптом, который уже на роутере (CI требует
    ровно один пакет на формат в `dist/`). Каталог поэтому вкомпилен внутрь пакета темы, а
    каталог с `.po` называется `i18n/`, не `po/` — именно переименование и отключает
    генерацию языковых пакетов. Попутно уходит скью: два пакета могли разъехаться, и тема
    со отстающим каталогом просто рисует новые строки по-английски, не сообщая ничего.

16. **msgid — ГЛОБАЛЬНОЕ имя, общее со всеми `luci-app` на роутере.**
    `action_translations` вызывает `load_catalog(lang, …)`, который сливает **все**
    `*.<lang>.lmo` из каталога в ОДИН словарь, а поиск возвращает первый архив с нужным
    хэшем — то есть **порядок readdir решает, чей перевод победит**. Переключатель раскладки
    рисовал «Максимум» на русском роутере: чей-то каталог переводит msgid `Top` как
    «максимум» — верно в диалоге про полосу пропускания, бред на переключателе раскладки.
    Лечение: свой **msgctxt** (`_(str, 'footstrap')`; po2lmo ключует по `ctxt\1msgid`), и
    тогда ключ ваш и только ваш. Обратная сторона правила: **хром и строки логина
    намеренно ОСТАВЛЯЮТ без контекста** — унаследовать перевод luci-base в тех ~40 языках,
    где своего каталога нет, это фича, а не баг. Ничто из этого не падает громко: `_()` без
    каталога — просто английский текст.

17. **`!important` ИНВЕРТИРУЕТ порядок слоёв.** `@layer` устроен так, что поздний слой бьёт
    ранний независимо от специфичности — значит правилу в `theme` **никогда** не нужен
    `!important`, чтобы победить `base`. Но среди `!important`-объявлений приоритет слоёв
    разворачивается: important в `base` бьёт important в `theme`. Из этого два следствия:
    (а) если правилу нужен флаг, чтобы победить ДРУГОЕ правило своей же темы — правило не в
    том слое, флаг тут не лечение; (б) законный `!important` бьётся только с тем, что не
    перебивается слоями вообще — инлайновым `style=` и **неслоёными** правилами (а
    неслоёное бьёт любой слой; `package-manager.js` инжектит именно такой `<style>`).

18. **`curl` НЕ входит в дефолтный образ OpenWrt.** Базовый образ несёт `uclient-fetch`;
    curl — отдельно ставящийся пакет. Самообновлятор темы жёстко требовал curl — и на
    стоковом роутере кнопка Update и бейдж обновления умирали с «cannot reach the GitHub
    release API» (воспроизводится переносом `/usr/bin/curl` в сторону). Правильный ответ —
    **fallback на `uclient-fetch`, а не рантайм-зависимость** ради одного API-вызова на
    устройстве с 8 MB флеша (`LUCI_DEPENDS` так и остаётся `+luci-base`). `jsonfilter` и
    `sha256sum` в базовом образе есть — им зависимость не нужна. И раз пакет ставится с
    `--allow-untrusted`, вся цепочка доверия — это ровно две вещи: проверенный TLS-канал
    (никаких `-k` / `--no-check-certificate`, в том числе «на второй попытке» — падение
    проверки и ЕСТЬ случай MITM) и sha256, который GitHub публикует для ассета.

**Свежая установка vs апгрейд решается МАРКЕР-ФАЙЛОМ.** Свежая установка вправе активировать
тему, апгрейд не вправе менять активную **никогда**. Переменная `$PKG_UPGRADE` для этого не
годится: **apk её не выставляет**, postinst тоже — на живом роутере guard был МЁРТВ (branch
брал только `dev-sync.sh`, который экспортирует её руками), и апгрейд, заставший пустой
`mediaurlbase`, активировал тему за спиной пользователя. Реальное состояние, различающее эти
два случая, — файл `/usr/share/luci-theme-footstrap/.installed`: он существует тогда и только
тогда, когда скрипт хоть раз доработал до конца. Пишется **последним** (упавший на середине
прогон остаётся «свежей установкой»), удаляется в `postrm` (переустановка снова свежая).

Про `sysauth.ut` — почему он обязателен и почему bootstrap'овский паттерн со скрытой формой
и view-модулем даёт пустую страницу без возможности залогиниться: docs/03.

> **Как это выглядит в footstrap.** `cascade.css` не лежит в git — он собирается из `styles/`
> (`build-css.sh`, docs/17), `mobile.css` нет: адаптив внутри той же сборки
> (`styles/theme/90-responsive.css`). Шапка/подвал разбиты на `partials/*.ut`.
> **Свой `sysauth.ut` ЕСТЬ и обязателен** (см. п. выше и docs/03).
> Клиентского JS четыре файла: `menu-footstrap.js` (единственный рендерер меню),
> `menu-footstrap-common.js` (bootstrap хрома) и модули `fs-*.js` — по одному на концерн
> (`fs-router`, `fs-chrome`, `fs-prefs`, `fs-widgets`, `fs-menutree`, `fs-sheets`,
> `fs-appearance`, `fs-update`), `fs-fit.js`
> (единственный движок «влезает ли ещё?») и `fs-select.js`. Файла с суффиксом `-top` нет
> ни одного — второго рендерера не существует.
> Запись в `luci.themes` **одна** (`Footstrap`): раскладка — это `:root[data-layout]`,
> клиентский переключатель, а не запись темы и не серверный выбор; режим и палитра — так же
> (док 04). `postrm` при этом всё равно удаляет все легаси-имена.
> Покрытие виджетов проверяется по `docs/gallery.html` — там отрисованы все классы, которые
> LuCI и сторонние `luci-app-*` вообще умеют выдавать; диффа с upstream bootstrap нет:
> правила стоковой `cascade.css` живут в `styles/base/` (`@layer base`) и поглощаются по
> одному, с проверкой через `cssdiff` (docs/17). В рантайме тема зависит только от `luci-base`.
