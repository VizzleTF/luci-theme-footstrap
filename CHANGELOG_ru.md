# История изменений

Значимые изменения `luci-theme-footstrap`, новые сверху. Формат —
[Keep a Changelog](https://keepachangelog.com/1.1.0/), коммиты —
[Conventional Commits](https://www.conventionalcommits.org/), версии —
[SemVer](https://semver.org/). Разделы, в фиксированном порядке: Добавлено,
Изменено, Устарело, Удалено, Исправлено, Безопасность, Производительность —
по одному каждого типа на релиз. Руководство по стилю и формату:
[docs/21-changelog-stil-i-format.md](docs/21-changelog-stil-i-format.md).

[CHANGELOG.md](CHANGELOG.md) — зеркало этого файла. Оба правятся одним коммитом.

Каждый коммит пишет в `[Unreleased]`. При выпуске тега заголовок переименовывается.

## [Unreleased]

### Добавлено

- **Портал для авторов сторонних `luci-app-*`, опубликованный на GitHub Pages.** Одна автономная
  страница, которую открывает автор приложения, чтобы увидеть что копировать, какой цветовой токен
  читать и чего не делать — сетка из 26 имён export-tier (живая под dark/palette/tint, клик копирует
  `var()`), каталог компонентов с вкладками Preview/HTML/`E()` и правила стилей с реальным багом за
  каждым. Она генерируется (как `cascade.css`) из уже существующих источников — токены разбираются из
  `02-tokens.css`, компоненты вырезаются из `docs/gallery.html`, реальная таблица стилей и её шрифты
  инлайнятся — поэтому ничего не копируется руками и ничего не рассинхронизируется.
- **«Fix my styles» — codemod в браузере, прямо в портале.** Вставляешь кусок приложения (CSS,
  `<style>`, копию из DevTools) — механические ошибки переписываются на месте: цветовые литералы в
  export-токены с литеральным fallback, `--warning-*`→`--warn-*`, приватные `--fs-*` в подходящий
  export-токен, лишний `!important`, зашкаливающий `z-index`, `<font color>` — а структурные, которые
  трогать автоматом нельзя, помечаются (инжект в `<head>`, `:root {}`, классы без префикса,
  `window.onload`, `prefers-color-scheme`, захардкоженная тема редактора, стоковые селекторы без
  скоупа). Каждый цвет сохраняет оригинал как fallback `var(--token, оригинал)`, поэтому переписывание
  никогда не меняет то, как приложение рендерится. Скормить можно тремя способами: вставить CSS
  напрямую, кинуть целиком дамп **Save Page As** (флагает `<style>` / `<link>` в `<head>`) или
  запустить однострочный консольный грабер на живом приложении — он собирает все инжектнутые в
  `<head>` листы, каждый inline `style=` внутри `#view` и каждый `<font color>` в буфер. Проверено на
  реальном куске `oc.css` из OpenClash.

### Изменено

- **Кнопка «Reset» в Appearance теперь называется «Reset to default».** Просто «Reset» не говорила,
  *к чему* сброс; кнопка сбрасывает переопределения этого браузера на сохранённое кнопкой «Save as
  default» умолчание для роутера — теперь ярлык называет эту цель.

### Исправлено

- **Наведение на страницу «Диагностика» больше не подсвечивает весь блок управления, а пустые
  таблицы — свою строку «нет данных»** (issue #5). Один общий тинт `.table .tr:hover` срабатывал на
  любой таблице, включая layout-таблицы, которые тема не собиралась делать интерактивными: строка
  ввода+кнопки в «Диагностике» — это один `<tr>` на всю карточку (без строки-заголовка, поэтому она
  никогда не помечается как таблица данных), и наведение подсвечивало весь блок; так же на hover
  загоралась и строка-заполнитель таблицы данных («нет активных аренд»). Теперь hover строк
  ограничен настоящими таблицами данных (по ячейке, заполнитель исключён) и строками данных
  конфиг-таблиц.
- **Бейдж устройства/MAC в блоке «Сеть» теперь следует шкале скругления вместо плоских 4px** (issue
  #5). Он сохранял захардкоженные 4px базового виджета, которые смотрелись квадратными рядом с
  `--fs-radius` карточки; теперь берёт токен и скругляется вместе с осью Rounding в Appearance.
  Ограничено блоком статуса — маленькие бейджи зон файрвола остаются плотными.
- **Чекбокс «Сохранить настройки» в диалоге подтверждения обновления прошивки больше не липнет к
  тексту** (issue #5). Диалог attendedsysupgrade рендерит `label.btn > input[type=checkbox]` + текст,
  а базовое правило, обнуляющее margin чекбокса, оставляло между ✓ и словами лишь пробел; margin у
  `.btn > input[type=checkbox]` возвращает отступ, не трогая тумблеры.
- **Неудачное «Save as default» теперь показывает видимое сообщение вместо молчаливого тултипа.** При
  отклонении rpc сохранения — чаще всего из-за истёкшей сессии — старый код прятал ошибку в атрибут
  `title`, который никто не наводит, и клик выглядел как «ничего не произошло»; теперь попап на месте
  (отсутствующего) статус-текста показывает «Could not save the default. Reload the page and try
  again.», а сырую ошибку rpc держит в тултипе для отладки. Это не покрывает *удалённый*
  `/etc/config/footstrap`: rpcd стейджит запись в сессии, а commit затем молча ничего не делает, не
  пересоздавая файл, и возвращает success (замерено на роутере) — этот файл принадлежит пакету, а
  read-сторона уже откатывается на встроенные умолчания.

## [0.9.0] — 2026-07-15

### Добавлено

- **В Appearance появилась кнопка «Save as default» — сохраняет текущий вид как значение по умолчанию
  для всего роутера, для любого браузера и устройства.** Раньше весь поповер жил только в
  localStorage браузера, поэтому второе устройство — или очищенный кэш — начинало со встроенных
  значений. Save записывает все восемь осей (layout, тема, палитра, обои, tint, accent, скругление,
  подменю) в `/etc/config/footstrap`; сервер проставляет их до первой отрисовки, и свежий браузер
  наследует их без вспышки. Запись идёт через ограниченный rpcd ACL, дающий сессии `uci`-запись
  только в конфиг `footstrap` — rpcd проверяет каждое имя, ни одно значение не попадает в шелл.
  **Браузер всегда главнее**: собственный выбор устройства, сохранённый явно, перекрывает значение по
  умолчанию в обе стороны (заданный роутером tint можно снова выключить), а «Reset» очищает локальные
  переопределения этого браузера и возвращает его на сохранённое значение. Роутер, обновлённый со
  старой верхней темы, по-прежнему читает миграционный ключ `luci.main.footstrap_layout` как запасной
  для раскладки.

### Изменено

- **Коробка логотипа в шапке теперь того же размера, что и остальные квадратные кнопки бара (34px).**
  Она была 30px, тогда как Appearance, Log out и свёрнутая пилюля «Refreshing» — все 34, единственный
  «неровный» квадрат в ряду правого кластера. Размер теперь один токен (`--fs-btn-size`), который
  читают все четыре, — ряд остаётся набором равных квадратов и настраивается в одном месте. Марка
  внутри коробки выросла с 17 до 20px, чтобы заполнить увеличенный квадрат.
- **Логотип в шапке теперь показывает марку OpenWrt из фавикона в рамке.** Акцентной плитки-
  градиента за логотипом больше нет; вместо неё простая рамка 1px — та же, что у квадратных кнопок
  в правом кластере (Appearance, Logout). Глиф — та самая марка, что показывает вкладка браузера.
  Его кольцо следует за режимом темы: тёмное на светлом UI, светлое на тёмном — так же, как SVG-
  фавикон следует за `prefers-color-scheme`, только логотип читает собственный флаг тёмного режима
  темы, а не media-запрос. Cyan-дуги фиксированы и читаются в обоих режимах.
- **Верхний бар теперь сворачивается по измерению на любой ширине, а индикатор «Refreshing»
  сжимается в иконку до переноса меню на вторую строку.** Раньше бар переключался на телефонную
  раскладку по жёсткому брейкпоинту 768px; теперь `fitChrome` (ResizeObserver) сначала ужимает пилюли
  меню, затем меняет пилюлю «Refreshing» на иконку-квадрат с рамкой — зелёную, серую на паузе, как
  кнопки Appearance и Logout, — освобождая ~56px, которых часто хватает, чтобы меню осталось на
  строке бренда, и переносит меню на вторую строку только если и этого мало. Порога 768px нет:
  влезает ли меню, зависит от числа секций роутера, а не от экрана. Раскладка-сайдбар сохраняет свой
  измеряемый телефонный бар (`data-narrow`), не тронута.

### Исправлено

- **Дропдаун секции прыгал к левому краю бара, как только верхнее меню переносилось на две строки.**
  Ниже старого брейкпоинта 768px бар откатывался на телефонную раскладку, прижимавшую каждый дропдаун
  к левому краю, так что перенесённое меню могло показать подменю Network под Status. Теперь бар
  измеряется на любой ширине, каждый дропдаун держится под своим пунктом, а существующий clamp не даёт
  ему вылезти за вьюпорт.

## [0.8.9] — 2026-07-15

### Добавлено

- **Гейт против всего этого класса багов: ни одно правило CSS не смеет матчиться на ЗНАЧЕНИЕ
  `data-title`** (`npm run css-i18n` плюс шаг в CI). Читать атрибут можно — так карточная таблица
  печатает подписи колонок, — но матчиться на него значит матчиться на переведённую и зависящую от
  рендера строку интерфейса, и отказ молчалив в обе стороны: мёртво на каждом незнакомом языке и мёртво везде, где собственный стиль
  применяет заголовок. Проверки на наличие
  (`[data-title]`) остаются разрешёнными.

### Изменено

- **`menu-footstrap-common.js` на 1676 строк разложен на модуль-на-концерн.** Файл разросся до семи
  несвязанных вещей сразу — оси Appearance, disclosure-примитивы, разрешение дерева меню, рендер
  хрома с замерами, SPA-роутер, защита от чужого CSS и самообновление, — а такой файл перестают
  читать: один и тот же `EDGE_GAP` был написан дважды, а UI обновления дотягивался до собственного
  refresh через глобал `window.__fsUpdateApply` просто потому, что шва не было. Разрезан на
  `fs-menutree` (путь ⇄ узел меню, порт `dispatcher.uc`), `fs-prefs` (оси и их localStorage),
  `fs-widgets` (disclosure-примитивы, seg/slider, размещение попапа), `fs-chrome` (mode-меню, табы,
  рельс, `fitShell`/`fitChrome`), `fs-router` (SPA-роутер), `fs-sheets` (защита от инжектнутого CSS),
  `fs-update` (`FS_VERSION`, проверка, установка в один клик) и `fs-appearance` (DOM поповера); в
  `menu-footstrap-common.js` остался только bootstrap. Поведение не изменилось — проверено на
  роутере: хром рендерится, SPA-навигация по-прежнему подменяет view на месте (без полной
  перезагрузки), Back работает, поповер собирает все девять групп, оси применяются, в консоли ноль
  ошибок; вывод jsmin остаётся token-identical на всех 13 файлах.
- **Модули композируются ВЫЗОВОМ, и граф проверяет сам рантайм.** `L.require` создаёт каждый модуль
  один раз синглтоном, поэтому модуль не может наследоваться от модуля (docs/11), и кидает
  `DependencyError` на цикле — так что общие половины (`fs-menutree`, `fs-prefs`) вынесены ВНИЗ
  графа, а не «дотягиваются» вбок: получился DAG, который рантайм и проверяет. Зависимости
  разрешаются через `Promise.all`, поэтому лишние файлы стоят раунд-трипов параллельно.
- **Храповик минифицированного JS поднят 47104 → 50176 Б — это и есть цена разбиения.** 46 621 →
  49 121 Б, +2 500 Б (+5.4 %): каждый модуль несёт свои прагмы и `return baseclass.extend({…})`, а
  каждый вызов через шов обрастает префиксом алиаса. uhttpd не жмёт, так что это байты по проводу —
  подъём осознанный размен на «ни одного файла длиннее ~600 строк», а не дрейф.

### Исправлено

- **Цвет зоны вылезал за скруглённый угол блока интерфейса** (issue #7). Network → Interfaces рисует
  зону инлайновым фоном на `.ifacebox-head`, и в `base` пара согласована — коробка 4px, голова 3px,
  скругляются вместе. Но `theme` поднял до `--fs-radius` (по умолчанию 10px) **только коробку**, и
  голова с квадратными углами 3px резала скругление насквозь. Теперь голова берёт радиус от коробки
  минус 1px бордера, внутрь которого вложена, — и они скругляются вместе при любом значении оси
  Rounding.
- **MAC-адрес по-прежнему рвался на две строки на любом неанглийском роутере** (issue #7). Тот
  самый `nowrap`, который должен был это чинить, матчился на
  `[data-title="MAC address"]` — а `data-title` LuCI заполняет из ЗАГОЛОВКА колонки, так что на
  русском роутере в ячейке стоит `MAC-адрес`, и правило не попадало никуда. Баг «починили», выпустили, а
  репортёр продолжал его видеть: фикс работал только на языке, на котором был написан. Привязано к
  колонке — перевод не может переставить колонки. Та же мёртвая-в-40-языках схема была в таблице
  аренд DHCP (DUID, список IPv6, имя хоста) и исправлена вместе с ней.
- **Правило в менеджере пакетов не попадало вообще никуда — ни на одном роутере, ни на одном
  языке.** Ячейка Description в карточном режиме матчилась на `[data-title="Description"]`, но LuCI
  строит ячейки этой таблицы из `innerText` заголовка, а наш собственный `text-transform: uppercase`
  на `.th` означает, что в атрибуте реально лежит `DESCRIPTION`. CSS темы переписывал строку, по
  которой матчился CSS темы, — и ячейка никогда не получала свою блочную вёрстку (замер: 0
  совпадений). Привязано к колонке; вёрстка наконец применяется.
- **Три гейта были нацелены на одно имя файла и замолчали бы.** `tools/axes.mjs` читал контракт
  Appearance из `menu-footstrap-common.js` по имени — ось, уехавшая куда-то ещё, не проверялась бы
  ничем; теперь он читает всё дерево `resources/`, что так устареть не может. Глобалы ESLint для
  алиасов `'require x as y'` были рукописным списком по файлам — ровно та форма, которая перестаёт
  покрывать следующий добавленный модуль, — и теперь выводятся из прагм самого файла, что заодно
  сохраняет способность `no-undef` поймать файл, который использует `prefs.`, забыв его
  затребовать. `Makefile` и `dev-sync.sh` штампуют `FS_VERSION` через `sed` по пути — оба переведены
  на `fs-update.js`; иначе поповер молча показывал бы «(dev)», а проверка обновлений перестала бы
  работать.
- **`css-orphans` объявлял живой селектор мёртвым CSS, потому что глушил ИМЯ.** Имена модулей
  игнорировались по имени, чтобы их `require`-прагмы не читались как классы, — и это ломается ровно
  тогда, когда модуль назван по разметке, которой владеет: `fs-appearance` — это и модуль, и id
  кнопки, которая его открывает, так что игнор прятал и настоящий `#fs-appearance`. Теперь гасятся
  ПОЗИЦИИ, в которых модуль упоминается (строка прагмы, `L.require('…')`), и позиции, в которых
  `fs-`-токен не класс (кастомные свойства `--fs-*`, атрибуты `data-fs-*`), — имя можно делить между
  модулем и его разметкой, не обманывая инструмент.

## [0.8.8] — 2026-07-14

### Исправлено
- **Страница пакетов на телефоне: `filtered / all / none` стояли по одному на строку.** Правило,
  раскладывающее заголовки контролов этой страницы, было написано как `.controls label` — и заблочило
  заодно три подписи радиокнопок, вложенные уровнем глубже. Теперь оно целится в собственный заголовок
  группы (`.controls > div > label`), и три варианта стоят в одну строку — в том числе на 320px.
- **Страница пакетов на телефоне: пейджер разъезжался на три строки, а карточный список пакетов
  по-прежнему печатал шапку колонок, которая ему уже не нужна.** Оба бага одной формы — правило,
  нацеленное на одно, попадает в другое, у которого просто совпало имя элемента.
  Пейджер (`«` / `Displaying 1-100 of 7677` / `»`) — это `<div class="pager">` внутри `.controls`, а
  телефонное правило, раскладывающее *подписанные группы контролов* этой страницы по вертикали,
  написано на `.controls > div`, — то есть заблочило и пейджер, и три его ребёнка встали по одному на
  строку: 97px высоты там, где хватает 43. Теперь `.pager` исключён.
  Шапка — та же история слоем выше: `pages/30-software.css` задаёт форму карточной строки через
  `#packages.fs-stacked .tr`, а **селектор с id в слое `page`** перебивает правило
  `theme/30-tables.css` — `.table.fs-stacked .tr.cbi-section-table-titles { display: none }`. Шапка
  колонок возвращалась на экран, где каждая ячейка и так печатает свою подпись. Обе строки-шапки из
  этого правила исключены.
  Десктоп не тронут: 0 расхождений вычисленных стилей на списке пакетов, обзоре и DHCP.
- **Состояние портов: на русском роутере порт с опущенным линком, через который шёл трафик, выталкивал
  свои цифры за пределы карточки и заводил их под соседнюю** (issue #7). Плитка — это две колонки
  (скорость слева, TX/RX справа), и вопрос «влезают ли они рядом» решался порогом `@container` в 158px.
  Порог — это **прокси для вопроса о контенте, и откалиброван он был на английском**: `no link` ≈ 45px
  против `нет соединения` ≈ 100px, а цифры по замыслу `nowrap` (`▲ 151.2 MiB` ≈ 85px против ≈ 35px,
  когда счётчики по нулям). Такой комбинации нужно ≈ 193px внутри карточки с контент-боксом 178 —
  порог не срабатывал, а grid не переносит: содержимое вылезало наружу. При нулях та же карточка
  влезала, что репортёр и описал.
  Теперь раскладка — переносящаяся flex-строка, и она задаёт настоящий вопрос сама: ячейки стоят рядом,
  пока влезают, и цифры уходят на свою строку, когда нет. Оба порога `@container` удалены — **их
  удаление и есть фикс**, а не побочный эффект. Без JS: обсервер (первое, что рассматривалось) должен
  был бы перемерять на каждом поллинге, ведь `29_ports.js` пересобирает плитки каждые 5 с, — чтобы
  вычислить то, что алгоритм раскладки уже знает.
  Три ловушки по дороге, каждая поймана замером, а не чтением: `flex-basis: 100%` на этой карточке не
  резолвится (у неё `container-type: inline-size`, то есть главный размер — не определённая длина;
  работает `width: 100%`); `margin-left: auto` Chrome учитывает при разбиении на строки, из-за чего
  цифры переносились даже там, где влезали; а перевод карточки с `grid` на `flex` разбудил
  `flex-direction: column`, который `base` всегда задавал каждому `.ifacebox` и который grid обесценивал
  — всё встало в колонку, пока ось не была указана явно.

  выше), а `padding-left` у `.spinning` был помечен, чтобы побить **их**. Первые два решаются поздним

- **Публичный гайд советовал сторонним авторам сломать собственные пакеты.** `docs/20` уверял, что

- **Четыре дока велели поставить `LUCI_MINIFY_JS:=0`** — это раздуло бы шипящийся JS втрое. Makefile

- **Около сорока комментариев в исходниках описывали код, которого больше нет, а некоторые — прямо

  в `Build/Compile` (он переехал в `Build/Prepare`), docstring `audit.py` рекламировал проверку
  баланса скобок в JS, которую намеренно убрали, а комментарий в uci-defaults говорил «удали
  маркер» там, где код его **пишет**.

### Безопасность
- **Каждый релизный пакет теперь подписан ed25519, и установщик с кнопкой «Обновить» отказываются
  ставить пакет без нашей подписи.** Одной sha256, которую установщик проверял и раньше, не
  хватает, и причина точная: GitHub *вычисляет* публикуемый дайджест из тех байтов, что были
  залиты. Любой, кто может подменить ассет релиза — достаточно утёкшего токена с правом записи,
  без всякого прогона CI, — получает пересчитанный под него дайджест, и контрольная сумма затем
  спокойно подтверждает пакет атакующего. Ключ подписи лежит в секретах CI, его нет ни в одной
  ветке и его нельзя вычитать обратно из GitHub, поэтому та же подмена валится на подписи:
  проверено на роутере живым скриптом (ассет подменён, дайджест пересчитан → sha256 проходит,
  `ERR: BAD SIGNATURE`). `usign` есть в любом образе OpenWrt (от него зависит `base-files`), так
  что подпись не стоит теме ни одной новой рантайм-зависимости, покрывает apk и ipk одним
  механизмом и — в отличие от доверия нашему ключу в `/etc/apk/keys` — не даёт на роутере никаких
  полномочий за пределами этого одного пакета. Обе проверки падают **закрыто**: нет дайджеста, нет
  ассета `.sig` или нет `usign` на устройстве — отказ. Присутствующую, но **неверную** подпись
  переопределить нельзя никогда.
- **CI отказывается публиковать релиз, который не может подписать, и отказывается от ключа, который
  роутеры отвергнут.** Публичная половина едет в пакете и вторым экземпляром вшита в `install.sh`
  (он запускается через `curl | sh`, до того как какой-либо пакет существует). Расхождение двух
  копий не ловится ни одним тестом — установщик просто отверг бы каждый релиз с `BAD SIGNATURE`,
  то есть отказ выглядел бы ровно как атака, — поэтому CI сверяет их на каждом прогоне, а релизная
  джоба перепроверяет каждый свежеподписанный пакет тем самым ключом, которым будет проверять
  роутер.

### Производительность
- **Загрузка страницы больше не порождает CGI-процесс ради пустого каталога переводов — минус 31 мс
  на каждой полной загрузке англоязычного роутера.** `<head>` синхронно грузил
  `admin/translations/<lang>`, и при `lang=en` тратил на это 31 мс (замер, пять прогонов), чтобы
  доставить **13 байт** — `window.TR={};`: английского каталога не существует, потому что msgid'ы и
  так английские. Платили за процесс, а не за данные. Теперь шаблон отдаёт те же 13 байт инлайном,
  когда каталога для языка нет, и сохраняет тег, когда он есть. Проверка зеркалит правило самого
  сервера (`*.<lang>.lmo` в `/usr/lib/lua/luci/i18n` — именно это глобит `load_catalog`), так что
  роутер, где английский каталог всё же стоит, получит тег; решение по имени языка молча выбросило бы
  его. Проверка падает **в сторону загрузки**: если она бросит исключение, тег остаётся, потому что
  отсутствие каталога делает каждый `_()` английским и ничего об этом не сообщает. `defer` отвергнут
  не по невнимательности: `footer.ut` выполняет `L.require('menu-footstrap')` инлайном, пока парсер
  ещё идёт, и `_()` из модуля обогнал бы отложенный `window.TR` — молча.
- **Страница логина избавилась от 17 копий 49-символьного селектора с `:has()` — 663 байта CSS.**
  Все правила цеплялись за `form:has(> .cbi-map input[name="luci_username"])` — в предположении, что
  разметка стоковая и назвать форму нельзя. Форма наша: её рендерит `sysauth.ut`, и теперь она несёт
  `class="fs-login"`. Оговорка аудита — что `ui.js` якобы перерисовывает форму логина в модалке
  истёкшей сессии — проверена и оказалась ложной: в `ui.js` нет ни одного `luci_username` и он вообще
  не строит форму логина, так что этим селекторам больше нечего было ловить. Вычисленные стили на
  живом роутере идентичны в светлой и тёмной теме (0 расхождений по всем элементам страницы).

### Изменено
- **Переключатель раскладки теперь «Сбоку» / «Сверху».**

### Исправлено
- **Кнопка «Обновить» ставила 6-килобайтный каталог перевода вместо темы.** В 0.8.4 в релизе
  появился второй пакет (`luci-i18n-footstrap-ru`), а `footstrap-selfupdate.sh` — в той версии,
  которая **уже лежит на роутере** — выбирает ассет так: `grep -E '\.apk$' | head -1`. GitHub отдаёт
  список ассетов отсортированным **по имени**, и `luci-i18n-footstrap-…` идёт раньше

  оставлял тему на старой версии и снова показывал бейдж — и так вечно, потому что скрипт, который
  выбирает не тот ассет, как раз и не обновляется. Починить его на роутере удалённо нельзя: что бы
  мы ни выложили, он запустит тот выборщик, который у него есть. Поэтому релиз снова несёт **один

  один пакет на формат. Если после «Обновить» появился русский, но версия осталась прежней, это тот
  самый случай: нажмите на 0.8.5 ещё раз, и тема установится.
- **Чужой пакет перехватывал переводы строк темы** — переключатель раскладки показывал «Максимум».
  LuCI отдаёт клиенту **один объединённый каталог** (`load_catalog()` читает все `*.<lang>.lmo` в
  `/usr/lib/lua/luci/i18n`, а поиск возвращает первый архив, где нашёлся хеш), так что msgid — это
  имя, общее со всеми `luci-app` на роутере, и победителя решает порядок чтения каталога: кто-то
  переводит msgid `Top` как «Максимум» — верно в диалоге про полосу пропускания и бессмысленно на
  переключателе раскладки. Теперь каждая подпись в поповере «Оформление» несёт контекст `footstrap`,
  и ключ становится только нашим. Строки хрома и логина намеренно оставлены без контекста (в ~40
  языках, для которых у темы нет своего каталога, они бесплатно получают корректный перевод из
  `luci-base`), и так же — System/Memory/Storage в надстройке обзора: она **сопоставляет** заголовки
  штатных секций и обязана разрешаться ровно так же, как `luci-mod-status`.

## [0.8.4] — 2026-07-14

### Изменено
- **`install.sh` теперь требует `jsonfilter`, а не откатывается на grep по ответу API.** Он входит
  в базовый образ OpenWrt и именно им читается sha256 ассета — единственная проверка целостности
  при `--allow-untrusted`. Так что запасной путь всё равно упирался в отказ «нет sha256 — установку
  отменяю». Упасть с одной понятной строкой лучше, чем через три шага с сообщением про безопасность.

### Исправлено
- **Строки самой темы оставались английскими на переведённом LuCI — пакет перевода никогда не
  попадал в релиз.** `po/ru/footstrap.po` полон уже несколько выпусков, и CI падает на пустом
  msgstr, но ни один `.lmo` не доезжал до роутера: OpenWrt SDK собирал `luci-i18n-footstrap-ru`,
  а маска `find` в сборочной джобе перечисляла только `luci-theme-footstrap-*` — и языковой пакет
  выбрасывался вместе с остальным `bin/`. Отчёт пришёл с полностью русского LuCI (issue #6), где в
  поповере «Оформление» значилось «Palette» / «Rounding» / «Cats», а переключатель раскладки —
  **«Максимум»**: это перевод msgid «Top» из `luci-base`. LuCI отдаёт клиенту ОДИН объединённый
  каталог (`load_catalog(lang, '/usr/lib/lua/luci/i18n')` читает все `.lmo` в каталоге), поэтому
  отсутствующий каталог не падает — его msgid тихо разрешаются по чужому или проваливаются в
  английский. `install.sh` и кнопка «Оформление → Обновить» теперь ставят языковые пакеты вместе с
  темой, а CI проверяет наличие ОБОИХ пакетов по имени: условие «каталог dist непустой» — ровно то,
  что позволяло восьми релизам уехать без перевода.
- **`install.sh` и самообновление могли поставить 6-килобайтный языковой пакет вместо темы.** Оба
  выбирали ассет релиза по расширению (`grep '\.apk$' | head -n1`), то есть по тому порядку, в
  котором GitHub перечислил ассеты. Пока в релизе был ровно один пакет, это было безобидно; как
  только рядом встали пакеты перевода — стало подбрасыванием монеты. Теперь оба сопоставляют по
  ИМЕНИ пакета (`/luci-theme-footstrap[-_]…`), и две копии этого матчера закреплены `@mirror`
  (`gh/asset-urls`) рядом с `fetch()` и списком разрешённых хостов — то же вынужденное дублирование,
  защищённое от расхождения по той же причине.
- **Языковой пакет версионируется вместе с темой.** `luci.mk` берёт их версию из `PKG_PO_VERSION`,
  который по умолчанию выводится из git или из mtime — а в сборке SDK нет `.git`, так что каждый
  прогон CI штамповал бы `0.<yymmdd>.<secs>`: версию, не связанную с релизом и разную при каждой
  пересборке одного и того же тега.

## [0.8.3] — 2026-07-14

### Добавлено
- **CI компилирует ucode-шаблоны.** Парсера у них не было нигде: `luci.mk` копирует их на роутер
  как есть, поэтому лишняя скобка в `header.ut` собиралась зелёной, уезжала в релиз, и у всех
  пользователей LuCI молча откатывался на другую тему. CI теперь собирает `ucode` из
  закреплённого коммита — та же дисциплина, что уже применяется к `jsmin.c`, и по той же
  причине — и гоняет по каждому `.ut` штатную для LuCI команду `ucode -T -c`.
- **CI валидирует rpcd-ACL как JSON.** rpcd пропускает файл, который не смог разобрать, и молчит
  — так что лишняя запятая отняла бы у всех пользователей бейдж обновления и кнопку Update без
  единого другого симптома.
- **SDK OpenWrt проверяется по контрольной сумме.** Два *линтера* были закреплены по коммиту и
  sha256, а тулчейн, который собирает уезжающий пользователям пакет, приезжал на доверии. Теперь
  сверяется опубликованный рядом `sha256sums`, а загрузка прибивает https на редиректах.

### Изменено
- **`build-css.sh` проверяет тот файл, который действительно пишет, и отказывается писать
  слишком маленький.** Проверка скобок и числа правил шла по *входу* squeeze-прохода — при том
  что именно squeeze способнее всего испортить стиль: он отслеживает строки, склеивает строки и
  выбрасывает последний `;`. А единственным гейтом на готовый файл была верхняя граница размера,
  так что любой способ получить *обрезанный* `cascade.css` проходил молча. Теперь число правил
  обязано пережить squeeze без изменений, а у файла есть не только потолок, но и пол.
- **Тег без секции в changelog теперь валит релиз.** `release-notes.sh` писал предупреждение в
  stderr и выходил с кодом 0, публикуя страницу релиза с текстом «See the CHANGELOG» для версии,
  о которой changelog никогда не слышал, — ровно та ошибка, от которой защищает правило «никогда
  не тегай первым», только зафиксированная навсегда и публично. Русское зеркало тоже обязательно.
- Модалка ошибки установки собирает сообщение текстовым узлом, а не через `innerHTML`: `luci.js`
  присваивает *голую строку* через `innerHTML` и делает текстовый узел только из массива, а
  попадает туда сырой stderr от `apk`/`opkg` — единственная строка в теме, которую не составляли
  ни тема, ни LuCI.
- `tools/fs-orphans.mjs` больше не считает *модуль* `fs-fit` неоформленным классом. Вечная ложная
  строчка «NEW» в отчёте — это и есть то, что приучает отчёт не читать.
- **`audit.py` печатал неверную строку для каждой находки за всё время.** Вырезая комментарий, он
  удалял и его переводы строк, а номера строк считаются по этой обрезанной копии — так что каждый
  `file:line` уезжал вверх ровно на столько строк, сколько комментария стояло над правилом. В
  дереве, где комментариев больше, чем кода, это большой сдвиг: блок фокуса, который он называл
  `30-forms.css:336`, на самом деле живёт на `:353`. Находка, указывающая не на ту строку, — это
  находка, которую пойдут «чинить» не в том правиле.

### Исправлено
- **Редактор ACE в приложениях (SSClash и любое другое, где есть ace.js) выглядел чёрным
  прямоугольником без текста и вылезал за границы вёрстки.** SPA-роутер при уходе со страницы
  УДАЛЯЛ из `<head>` каждый `<style>`, который туда добавила вьюха, — верное решение для блоба
  файлового менеджера (ниже) и неверное для CSS, который его автор не может вернуть. ACE
  импортирует `ace_editor.css` (14 КБ: абсолютно позиционированные слои, гаттер, строки) ОДИН раз
  на документ, на этапе eval модуля, поэтому повторный рендер его не переинжектит — а темы и моды,
  подгружаемые под каждый редактор, возвращаются. Измерено на роутере: SSClash → Configuration,
  SPA-переход на Log и обратно — тема красит редактор чёрным, структура не возвращается, а
  непозиционированные слои раздувают страницу до 2 007 346 px в высоту. Удаление CSS было
  необратимым, поэтому роутер больше не удаляет ничего: лист, который может попасть только в
  виджеты своего же приложения (`.ace_*`, `.cpu-status-view-mode-entry` стокового обзора), на
  чужой странице не красит ничего и просто остаётся, а SPA-навигация по таким приложениям работает.
- **Приложения со своей тёмной темой рисовали на тёмной странице свои СВЕТЛЫЕ стили.** Приложению
  надо как-то узнать, тёмная ли тема, и обзор экосистемы LuCI нашёл три диалекта этого вопроса:
  `data-theme="dark"` на `:root` (у `luci-app-justclash` на нём висит 21 правило), бутстраповский
  `data-bs-theme` (его первым читает `luci-app-ssclash`) и яркость фона `body` (OpenClash, passwall
  и запасной путь ssclash). Тема стампила только свой `data-darkmode`, поэтому все тёмные правила
  justclash были мертвы. Теперь стампятся все три имени одного факта: `data-darkmode` остаётся тем,
  что читает CSS самой темы, два других — исходящая совместимость (как экспортный тир
  `--*-color-*`), а `tools/axes.mjs` роняет сборку, если их прочитает правило из `styles/` или если
  пре-paint в шаблоне разойдётся с живым применителем.
- **Фон `body` теперь доказуемо непрозрачный — именно его читают все сниферы тёмной темы.**
  OpenClash, passwall и ssclash решают «светло или темно» по яркости
  `getComputedStyle(document.body).backgroundColor`, причём регулярка OpenClash даже не матчит
  `rgba(0, 0, 0, 0)` — то есть прозрачный `body` заставляет его решить «светлая» и перекрасить
  тёмную страницу в свою светлую палитру (а `data-darkmode` он потом сам пишет нам в `:root`).
  Увести фон на `:root` или добавить альфу — ровно этот тихий сценарий, поэтому
  `tools/export-tier.mjs` проверяет непрозрачность и правильную сторону яркости по всей матрице
  палитра × режим × тинт.
- **Приложение, инжектящее свой CSS на каждый рендер, больше не копит копии.** `luci-app-podkop`
  вешает 4 КБ `<style>` в `<head>` прямо из `render()` без всякой защиты, `luci-app-mosdns` так же
  переприкрепляет три `<link>` CodeMirror; после отказа от свипа каждый SPA-визит оставлял ещё одну
  копию. Удаление байт-идентичного дубля — единственное удаление, которое никого не ломает (правила
  никуда не деваются, и библиотека по-прежнему находит свой лист), и теперь это единственное, что
  роутер удаляет.
- **CSS вьюхи по-прежнему не утекает на следующую страницу — и теперь это касается и `<link>`.**
  `luci-app-banip` и `luci-app-adblock` на eval модуля вешают в `<head>` `<link>` на `custom.css`,
  который стилит `.cbi-input-text` / `.cbi-input-select` — стоковые виджеты, на всех страницах, вне
  слоёв. Старый свип смотрел только на `<style>`, так что это утекало молча. Вместо свипа — проверка,
  а не список: лист *инвазивен*, если может покрасить не свою страницу. Это либо голый селектор
  (`h4`, `svg text`, `div > label + select`, `:root { color-scheme: … }`), либо селектор, целиком
  состоящий из имён, которые стилит сама тема, и не имеющий ничего собственного, чем он был бы
  приколот к разметке приложения. Обе формы матчат стоковые виджеты на любой странице и, будучи вне
  слоёв, перебивают все каскадные слои. Множество имён читается из `cascade.css` в рантайме, поэтому
  оно следует за темой, а не расходится с ней. Документ с таким листом отработан: следующий переход
  делается НАСТОЯЩЕЙ перезагрузкой — ровно то, что стоковый LuCI делает на каждой ссылке.
  Два исключения не дают правилу штрафовать невиновных, и оба выверены на живых приложениях:
  стоковый класс, **приколотый** собственным именем приложения (`#cbi-podkop-section >
  .cbi-section-remove`, `.bandix-table th.sortable.active`), без его разметки не сматчится — но
  аргумент `:not()` пином не считается, поэтому `.cbi-button-save:not(.custom-save-button)` из
  `luci-app-filemanager` по-прежнему инвазивен; а голый селектор, объявляющий только кастомные
  свойства, которых тема не читает (`:root { --app-temp-status-temp: … }`), красить попросту нечем.
  Проверено на восьми приложениях, стоящих на дев-роутере: ACE/ssclash, podkop, CPU-инклюд обзора и
  hex-редактор сохраняют SPA-навигацию, а `luci-app-filemanager` (`.cbi-button-save`), стоковый
  `luci-app-openvpn` (`h4 { white-space: nowrap }`), `luci-app-bandix` (`.error`),
  `luci-app-wrtbwmon` (`div > label + select`) и `luci-app-temp-status` (`svg text { fill }`) уходят
  полной загрузкой. Save/Apply/Reset на странице System после визита в файловый менеджер на месте.
  Замер: 0.3 мс на переход.
- **Два быстрых клика могли оставить вас на одной странице, а показать другую — и её поллер
  жил дальше вечно.** При ПЕРВОМ визите рендер страницы делает сам `require()` SPA-роутера, то
  есть отменить его нечем: клик по Firewall (не в кэше — модуль плюс его `load()`-RPC, на
  медленном канале это секунды), клик по Wireless через 100 мс — и роутер флашит очередь опроса
  ДО того, как Firewall успел добавить свой поллер. Firewall затем рисуется в `#view`, который
  уже принадлежит Wireless, и регистрирует поллер, до которого флашу уже не дотянуться.
  Воспроизведено на живом роутере с добавленной задержкой 1.2 с: URL, заголовок, меню и
  `body[data-page]` показывали System, а на экране был редактор зон Firewall. Перекрытый первый
  рендер теперь обнаруживается и отменяется — текущая страница перерисовывается, что заодно
  убивает осиротевший поллер.
- **Клик по странице, на которой вы уже находитесь, больше не убивает кнопку «Назад».** Роутер
  писал запись в историю безусловно, поэтому клик по активному пункту меню добавлял дубликат;
  «Назад» затем вызывал `popstate`, видел тот же путь и корректно не делал ничего — по разу на
  каждый лишний клик. Повторная навигация на текущий URL теперь заменяет запись, как это делает
  полная загрузка страницы.
- **Кнопка Update могла зависнуть до перезагрузки роутера.** Воркер, убитый посреди `apk add`
  (OOM на 128-мегабайтной коробке — а apk и есть прожорливая часть), навсегда оставлял
  `status=RUNNING` и свою staged-копию, а стоявшая перед локом проверка отвечала `RUNNING` на
  каждый следующий клик — клиент опрашивал все свои 300 с и писал «timed out waiting for the
  installer», и так всегда. Хуже того, механизм перехвата протухшего лока, написанный ровно под
  этот случай, не мог сработать никогда: проверка возвращалась раньше. Теперь решает только
  атомарный `mkdir`-лок — для этого он и был.
- **Клавиатура и скринридер снова могут проследить за навигацией.** Каждая SPA-навигация
  пересобирает меню, поэтому `<a>`, только что нажатая по Enter, исчезала из документа, фокус
  падал на `<body>` (следующий Tab начинал заново со skip-ссылки), и ничто не сообщало, что
  страница вообще сменилась. Теперь фокус переходит на `<main>` (у него уже был `tabindex="-1"`
  ради skip-ссылки), а заголовок новой страницы проговаривается через вежливый live-регион.
- **Документного `<h1>` не было в дереве доступности вообще.** Он прятался атрибутом `hidden`,
  то есть `display: none`, а это убирает элемент для ассистивных технологий так же полно, как и
  для глаза — так что структура заголовков, ради починки которой `<h1>` и добавляли, всё равно
  начиналась с `<h2>` внутри view, а синхронизация заголовка в роутере обновляла узел, который
  никто не мог прочитать. Теперь он клипается (`.fs-sr`) — тем же приёмом, что и skip-ссылка.
  Проверено по ARIA-снимку Chrome на живом роутере.
- **Меню нигде не говорило «вы здесь».** Активный пункт и активная вкладка несли только
  CSS-класс; теперь они несут `aria-current="page"`. Иконки и шевроны, которые рисует JS,
  получили `aria-hidden="true"` — как и все SVG в шаблонах.
- **Форма на Lua-CBI не показывала красную рамку у невалидного поля.** `styles/base` её объявляет
  для `.cbi-value-error input`, но сокращённое свойство темы `input { border: 1px solid … }` в
  более позднем слое затирает longhand независимо от специфичности — и поле рисовалось серым.
  Современный путь `.cbi-input-invalid` работал; терял подсказку только `luci-compat`, то есть
  каждое стороннее приложение, всё ещё сидящее на Lua-CBI. Измерено, а не выведено: серый
  `#d0d7de` до, danger-красный после.
- **Четыре типа input рисовались стоковыми белыми коробками с радиусом 3px** рядом с
  тематизированными полями в той же форме: `color`, `datetime-local`, `month` и `week` не были в
  списке типов темы, а пропущенный тип откатывается не в «без стилей», а в `base`.

### Безопасность
- **Самообновление ставило пакет без проверки sha256 всякий раз, когда не могло её найти.**
  Проверка была `if [ -n "$digest" ]` без `else`: переименование поля на стороне GitHub,
  переставший резолвиться предикат `jsonfilter` или отсутствие самого `jsonfilter` оставляли
  дайджест пустым — и пакет ставился с `--allow-untrusted` вообще без контроля целостности, с
  отчётом об успехе. Половина цепочки доверия из двух звеньев не может быть необязательной:
  отсутствующий дайджест теперь означает отказ. `install.sh` тоже отказывает
  (`FOOTSTRAP_ALLOW_UNVERIFIED=1` — осознанный ручной обход).
- **`__run` — привилегированная точка входа воркера — был достижим по RPC.** ACL `file.exec` в
  rpcd матчит *путь* команды, `params` свободны, поэтому любая сессия с этим ACL могла вызвать
  скрипт самообновления с `__run` напрямую, а это запускало установку в foreground и **без
  взятия лока**: два параллельных `apk add` одного пакета — ровно та гонка, ради которой лок и
  написан, — причём rpcd убивал один из них по своему 30-секундному таймауту, возможно посреди
  установки. Теперь ветка работает, только если скрипт вызван как staged-копия воркера.
- **Динамический загрузчик оставался на усмотрение вызывающего.** rpcd передаёт процессу и
  окружение, которым управляет вызывающий: `PATH` был закреплён, а вот `LD_PRELOAD` /
  `LD_LIBRARY_PATH` для `/bin/sh` — это выполнение произвольного кода от root для любого
  обладателя ACL, и прокси-переменные перенаправили бы загрузку. Всё это теперь сбрасывается.
- **Релизный токен больше не выдаётся каждому pull request.** `permissions: contents: write`
  стоял на уровне всего workflow, то есть был доступен `npm ci` в job'е линтеров — а значит
  lifecycle-скриптам каждой dev-зависимости — на прогонах `pull_request`. Теперь его объявляет
  только job релиза.

## [0.8.2] — 2026-07-13

### Изменено
- **Лицензионная позиция зафиксирована — в README и в Makefile.** Тема под Apache-2.0, и это **не**
  свободный выбор: `styles/base/` начинался как форк `cascade.css` из luci-theme-bootstrap,
  ucode-шаблоны производны от шаблонов LuCI, часть JS-хелперов скопирована из LuCI дословно — всё это
  Apache-2.0, чьи notices обязаны ехать вместе с кодом. (GPLv2 даже недоступна: патентная оговорка и
  indemnity в Apache-2.0 — это «дополнительные ограничения», которые GPLv2 запрещает. GPLv3 была бы
  законна, но стоила бы теме места в фиде LuCI и заставила бы вендоров прошивок её избегать — ради
  копилефта, который почти ничего не даёт коду, отдаваемому браузеру исходником.) Встроенные шрифты
  под неё **не** попадают: они под SIL OFL 1.1, и теперь об этом сказано.

### Исправлено
- **Встроенные шрифты распространялись без своей лицензии.** Manrope и JetBrains Mono — под SIL Open
  Font License 1.1, а §2 OFL требует, чтобы **каждая копия** Font Software несла копирайт **и** текст
  лицензии. Тема отгружала девять `.woff2` и ни того, ни другого — и не могла нести их внутри самих
  шрифтов: это сабсеты по unicode-range, а сабсеттер вырезает лицензию из name-таблицы шрифта
  (проверено: копирайт уцелел, поле лицензии — нет). Теперь рядом с ними на роутер едет
  `fonts/OFL.txt`.
- **Метаданные лицензии в пакете указывали в пустоту, а теперь пакет везёт то, что объявляет.**
  `PKG_LICENSE_FILES` резолвится относительно `$(PKG_BUILD_DIR)`, куда `luci.mk` кладёт только
  `src/ luasrc/ htdocs/ root/ ucode/ po/`, а CI rsync'ит в SDK лишь каталог пакета — так что корневой
  `LICENSE` был недостижим ниоткуда. `Build/Prepare` его копирует, а `PKG_LICENSE` стал честным
  `Apache-2.0 OFL-1.1`: пакет действительно несёт два разных корпуса кода. Две копии Apache-текста
  (корень — для GitHub, пакет — для сборки) закреплены побайтно через `npm run mirror`.
- **CSS, впрыснутый одним view, больше не таскается за вами по всем следующим страницам.** View может
  при отрисовке впрыснуть `<style>` в `<head>` — так делает `luci-app-filemanager`, — и при полной
  загрузке страницы этот стиль умирает вместе с документом, то есть влияет только на ту страницу,
  которая его попросила. SPA-навигация документ не перезагружает, поэтому он оставался в `<head>`
  навсегда. И это не косметика: блоб файлового менеджера содержит
  `.cbi-button-apply, .cbi-button-reset, .cbi-button-save { display: none !important }` (он прячет
  стоковые кнопки, потому что у него свои), а будучи **безслойным** и с `!important`, он перебивает
  любой каскадный слой. Измерено на роутере: один раз зайти в файловый менеджер, потом на System —
  **Save и Reset исчезают** и не возвращаются до жёсткой перезагрузки, то есть любая страница настроек
  после этого несохраняема. Теперь роутер сметает их при навигации — ровно так же, как уже сметает
  поллеры уходящего view, его `setInterval`-ы и открытые модалки: документ возвращается в то
  состояние, в котором его оставила бы свежая загрузка. Собственный `<style>` шелла, отданный
  сервером, помечен `data-fs-shell` и сохраняется — эти два случая различаются, а не угадываются.
  Проверено: повторный заход в файловый менеджер теперь отрисовывается ровно так же, как его полная
  перезагрузка.

  так объявлял на hover. Подписи у них `--fs-text`, а **не** цвет роли: текст цвета C на полупрозрачном
  тинте того же C — это ошибка, о которой проект прямо пишет, что выучил её дорогой ценой, и axe поймал
  её сразу (accent на accent-soft: 4.25:1, провал AA). Роль несут заливка и рамка; подписи достаточно
  быть читаемой.
- **Hover-подсветка меняет направление в зависимости от режима, и это исправление WCAG, а не украшение.**
  `filter` перекрашивает не только заливку, но и **текст**, а сплошная кнопка в светлой теме — это
  насыщенная заливка с **белыми** чернилами, которые ярче уже не станут. Осветление только сокращает
  разрыв: на `--fs-accent` белый текст даёт 5.19:1 в покое, старый `brightness(1.08)` ронял его до
  4.59:1, а подсветка, которую **реально видно** (1.15), — до **4.08:1, то есть провал, создаваемый
  самим наведением**. Измерено по отрисованным пикселям, а не посчитано. Поэтому светлый режим теперь
  затемняет (0.90 → 6.16:1: и сигнал сильнее, и контраст лучше), а тёмный, где заливка светлая, а
  чернила тёмные, — осветляет (1.15 → ~8:1). Оба говорят одно: кнопка уходит от фона страницы.
- **`install.sh` и `footstrap-selfupdate.sh` — закреплённые зеркала друг друга там, где обязаны
  ими быть.** Общий файл невозможен: установщик — это `curl | sh`, и он работает *до* того, как
  появится пакет с библиотекой; при этом оба обязаны качать по проверенному каналу, пинить хост
  ассета и сверять sha256. Это дублирование **уже разошлось**: у двух `fetch()` был разный порядок
  бэкендов, один не давал своему первому по очереди инструменту (`uclient-fetch`, на OpenWrt)
  *никакого таймаута*, а у другого не хватало пина https-редиректа на ветке `wget`. И никто ничего не
  сказал — именно потому, что разошедшаяся копия перестаёт выглядеть дублем. Теперь они побайтно
  одинаковы и закреплены `@mirror`, так что разойтись снова не смогут.
- **Апстрим-коммит, к которому пиньятся заимствованные сборочные тулы, живёт в одном файле.**
  `jsmin.c` (который CI компилирует и гоняет как гейт, доказывающий безопасность отгружаемого JS) и
  `i18n-scan.pl` (решающий, полон ли перевод) пиньились одним и тем же SHA в двух местах, каждое с
  комментарием «бампайте вместе», и ничто их вместе не держало. Оба теперь читают
  `luci-upstream.pin`.
- **Оси Tint и Accent — одна функция.** Их разделяло сорок почти одинаковых строк: та же валидация
  1–360, тот же «0 — это выкл», тот же clamp и то же несущее правило порядка (сначала custom-property,
  потом атрибут, иначе свежая загрузка рисует один кадр с прежним оттенком). Такое правило чинят
  ровно в одной копии из двух. Остальные семь осей Appearance намеренно не тронуты: у каждой есть
  настоящая особенность, под которую таблице пришлось бы отращивать опцию.
- **Два галерейных гейта делят один каркас.** `a11y-gallery.mjs` и `export-tier.mjs` несли каждый
  свою копию «собрать CSS, отдать галерею, проштамповать оси Appearance на `:root`» — и последняя
  часть была *четвёртой и пятой* копией правил, живущих ещё и в JS темы, и в `head.ut`. Гейт, который
  продолжает проверять старую форму, продолжает быть зелёным — а это хуже, чем отсутствие гейта.
- **`dev-sync.sh` деплоит resource-JS по маске, а не по именам.** Он перечислял четыре файла
  поимённо, поэтому пятый уехал бы в пакет (luci.mk копирует `htdocs/` целиком) и молча не попал бы
  на dev-роутер — то есть был бы впервые проверен уже после релиза. У deploy-скилла был тот же баг в
  худшей форме: он умел мапить `root/*` на роутер, но его поиск файлов никогда ему такого не
  отдавал — то есть правка бэкенда самообновления или его ACL с последующим деплоем не делала
  **ничего**, молча.
- **`fs-fit.js` теперь действительно владеет коалесингом кадров, как и утверждала документация.** Он
  экспортировал `schedule()` (который гоняет *все* фиттеры), но не давал способа забатчить одиночный
  колбэк — поэтому три вызывающих места вручную переписали одни и те же пять строк, а ещё два — один
  и тот же фильтр мутаций. Теперь это общие примитивы (`fit.frame`, `fit.touches`). Edge-clamp
  выпадашки сохранил собственный rAF-хэндл на каждый `<li>`: ему нужно **отменять** запланированное
  измерение, чего общий коалесер с одним флагом выразить не может.
- **README описывал продукт, которого больше нет.** Он предлагал «две раскладки
  (`FootstrapSidebar` / `FootstrapOnTop`), переключаемые в настройках LuCI» и «три палитры», одна из
  которых давно стала отдельной осью обоев. Тема в настройках одна; раскладка, палитра, обои, тинт,
  акцент, скругление и поведение подменю — всё это настройки браузера в попапе Appearance, и ни одна
  из них не была задокументирована. Самообновление не упоминалось вовсе.
- **Отступ от края экрана, которому подчиняются оба вручную позиционируемых попапа, — одна
  константа.** Попап Appearance и edge-clamp меню писали каждый свою восьмёрку.
- **Документация описывала кодовую базу, которая давно ушла вперёд.** Сплошная проверка всех
  проверяемых утверждений в `CLAUDE.md`, README, `docs/`, скиллах и комментариях кода нашла ~40
  ложных. Несущие: специфичность оверрайда сайдбара указывалась как `0,3,0` под медиазапросом
  `768px` (на деле `0,4,0` под `521px` + `:not([data-narrow])` — цифра осталась от версии без
  `:not()`); девять комментариев ссылались на `theme/20-shell-sidebar.css` — файл, которого нет;
  `FS_CSS_BUDGET` в двух местах описан как 124 КБ, а он 115 КБ; бюджет шрифтов в CI описан как
  100 КБ, а он 70 КБ (по документу можно было бы незаметно нарастить 33 КБ шрифтов); цифры про
  комментарии и минификацию JS устарели вдвое; часть комментариев всё ещё описывала карточный стек
  таблиц как контейнерный запрос, который заменён измерением. (`docs/16` и `docs/18` — датированные
  срезы аудита, они оставлены как история и не переписывались.) Лгущий комментарий хуже
  отсутствующего.
- **Геометрия шелла — три токена вместо шести копий трёх чисел.** `--fs-sidebar-w`, `--fs-rail-w` и
  `--fs-content-min` живут теперь в `02-tokens.css`; стиль раскладывает по ним сайдбар, а
  `menu-footstrap-common.js` читает их обратно, чтобы решить, читаемо ли то, что осталось контенту. В
  JS лежали собственные `SIDEBAR_W = 224, RAIL_W = 68, CONTENT_MIN = 500` против голых литералов в CSS,
  так что сужение рельса в стиле оставило бы измерение вычитать старую ширину — и ни один гейт сборки
  этого бы не заметил. В `20-shell.css` вдобавок упоминался токен `--fs-content-min`, которого не
  существовало.
- **Пакет объявляет собственного мейнтейнера и домашнюю страницу.** Без `LUCI_MAINTAINER`/`LUCI_URL`
  их подставлял `luci.mk`, и собранный пакет заявлял, что его сопровождает OpenWrt LuCI community. В
  репозитории появился и текст `LICENSE`, которого раньше не было, — но `PKG_LICENSE_FILES` пакет
  намеренно **не** задаёт: он резолвится относительно build-каталога, а `luci.mk` кладёт туда только
  `src/ luasrc/ htdocs/ root/ ucode/ po/`. Указывать в метаданных на файл, которого в сборке нет, хуже,
  чем не указывать вовсе.
- **Линтеры проверяли только те правила, которые кто-то не забыл перечислить.** `eslint.config.mjs`
  никогда не подключал `eslint:recommended`, поэтому `no-dupe-keys`, `no-unreachable`,
  `no-duplicate-case`, `no-prototype-builtins`, `getter-return`, `no-async-promise-executor` и ещё
  около тридцати бесплатных correctness-правил были просто выключены; у stylelint не хватало проверки
  грамматики значений (`declaration-property-value-no-unknown`) — единственного, что способно поймать
  декларацию, невалидную на этапе computed-value и потому исчезающую *молча*, ровно тот режим отказа,
  ради которого выпиливали компонентные мосты `--*-rgb`. Оба набора нашли на текущем дереве **ноль**
  нарушений: сегодня они не стоят ничего, а следующую ошибку поймают даром.

### Исправлено
- **Любой view, использующий `<a href="#">` для собственных контролов, терял состояние на каждом
  клике** (issue #3, `luci-app-filemanager`). Chrome шлёт `popstate` при навигации по *фрагменту* в
  том же документе — поэтому клик по такой ссылке внутри view приходил в SPA-роутер так, будто
  пользователь нажал «Назад». Роутер заново выполнял навигацию на путь, который и так на экране, а это
  **пере-инстанцирует view** — и отменял то, что клик только что сделал, через один оборот цикла
  событий. Полоса вкладок файлового менеджера — это четыре `<a href="#">`, чей обработчик не зовёт
  `preventDefault`, поэтому переключение на Editor, Settings или Help происходило **и тут же
  откатывалось**, а приложением было невозможно пользоваться. Прослежено на роутере: `popstate` →
  в `#view` появляется совершенно новый контейнер. Две ошибки «Не удалось отобразить список файлов» из
  того же отчёта — этот же баг с другой стороны: каждый внезапный перерендер заново запускает
  `render()` приложения, а его список файлов гоняется наперегонки со вставкой DOM, от которой зависит.
  Смена фрагмента — не навигация: роутер теперь сравнивает **путь** и не вмешивается, если сдвинулся
  только фрагмент. Back/forward между настоящими путями по-прежнему идут через SPA, без единой полной
  перезагрузки.
- **Страница входа тащила весь хром — сайдбар, меню и футер — вокруг формы, у которой единственный
  элемент управления это поле пароля.** Тема не везла `sysauth.ut`, поэтому LuCI брал generic-шаблон, а
  тот подключает header **без** `blank_page` (у luci-theme-bootstrap свой шаблон, и этой проблемы у
  него нет). Теперь свой есть и у нас. И это намеренно **не** копия bootstrap: тот прячет форму в
  `<section hidden>` и раскрывает её view-модулем, а эту схему тема уже пробовала — получилась пустая
  страница без возможности войти: view стартует до появления сессии, его RPC отвечают «Access denied»,
  промис реджектится, `render()` не выполняется. Форму рендерит сервер — она работает и с выключенным
  JS, и её нельзя сломать отвалившимся промисом.
- **Страница входа игнорировала обои «Коты».** Тёмный режим, палитра и тинт до неё доезжали (они висят
  на `body`), а обои рисуются на `.fs-shell`, которого у страницы без хрома нет — так что единственный
  экран, который видишь раньше всего остального, был единственным, не совпадавшим с темой.
- **Таблица данных без `id` теряла отступы ячеек, моношрифт и подсветку строки.** `[id]` и `.fs-dt` —
  два имени для одного («это таблица данных, а не key/value-врезка»), и написаны они были двумя
  списками селекторов с **разной** специфичностью: `.cbi-section .table[id] .td` это (0,4,0), а
  `table.fs-dt .td` всего (0,2,1) — и проигрывает key/value-умолчанию с (0,3,0). Таблица с `id` жила
  нормально, а опознанная только JS-тегом сохраняла key/value-отступ (`10px 16px 10px 0` — прижатый
  влево, что верно для колонки-подписи и неверно для ячейки данных). На живом роутере: **Status →
  Routing** упирал каждую ячейку в левый край таблицы. Теперь оба имени — один `:is([id], .fs-dt)`, и
  разойтись они больше не могут.
- **Набор с клавиатуры в открытой выпадашке теперь прыгает к нужному пункту, как в нативном
  `<select>`.** Откройте Country Code, наберите «ru» — нативный select подсветит «RU - Russian Federation»;
  так и выбирают один пункт из 248. Тема подменяет нативные select'ы на стилизованный `ui.Dropdown`
  (нативный попап не стилизуется), а в `ui.Dropdown` **поиска по буквам нет вообще** — в bootstrap он
  как будто есть только потому, что там это поле остаётся настоящим `<select>`. Type-ahead реализован
  для любого `.cbi-dropdown`, включая те, что рисует сам LuCI: буфер сбрасывается по паузе, повтор
  одной буквы циклит по пунктам на неё, а подпись сверяется раньше значения — так что находят и «ru», и
  «russ». Enter коммитит, как и раньше.
- **Строка копирайта в футере была прижата влево.** `text-align: center` её не центрировал: base делал
  футер flex-строкой с `justify-content: space-between` — наследие двухколоночного футера — а с
  единственным `<span>`, который везёт эта тема, space-between паркует его в начало.
- **Кнопка «Refresh Channels» (Status → Channel Analysis) стояла вплотную к секции под ней** и читалась
  как часть той карточки, а не как действие уровня страницы. У `.cbi-title-buttons` не было нижнего
  отступа.
- **Контрол «Submenus» в поповере Appearance игнорировал переключение раскладки.** Переключатель
  аккордеона бессмысленен в верхней раскладке (её разделы — выпадашки по наведению, они и так
  взаимоисключающие), и его убирали через `if (currentLayout() !== 'top')` вокруг группы. Но поповер
  строится **один раз**, в `init()` — поэтому эта ветка замораживала контрол по той раскладке, в
  которой *загрузилась страница*: переключаешься на верхнюю панель — он остаётся на экране;
  загрузился в верхней панели и переключился на сайдбар — он не появляется никогда. Теперь он
  строится всегда и прячется в CSS по `:root[data-layout="top"]` — это собственное правило темы
  (переключение раскладки ничего не перерисовывает, CSS морфит хром), поэтому он корректен и при
  загрузке, и при переключении, и вообще без состояния в JS.
- **В окне шириной ~768–779 px меню и стиль расходились в том, что такое шапка.** CSS давно ушёл с
  брейкпоинта 768 px: сайдбар уступает, когда *колонка контента* упала бы ниже минимума, измеренного
  от реального выреза сайдбара (`data-narrow`). А `flyoutMode()` в меню всё ещё спрашивал
  `matchMedia('(max-width: 767px)')`, и его комментарий ссылался на файл, которого больше нет.
  Измерено на живом роутере при 770 и 775 px: шапка рисовалась панелью во всю ширину, а меню считало
  себя вертикальным аккордеоном — секция раскрывалась развёрнутой *внутри* панели, клик снаружи и
  Escape её не закрывали, а edge-clamp выпадашки отказывался её ставить. Хуже того, `data-narrow`
  никто не наблюдал, поэтому протаскивание окна через границу не запускало смену режима вообще.
  Теперь оба читают один и тот же атрибут.
- **Веса колонок (`col-1`…`col-10`) не доходили до таблицы, скарточившейся выше телефонного
  тира** — вторая половина разъехавшегося карточного контракта. Теперь они без гарда, и это не
  костыль: `flex` инертен на всём, кроме flex-элемента, а ячейка становится им ровно
  тогда, когда таблица скарточилась — *любым* из двух механизмов. Одна копия вместо двадцати правил
  под двумя гардами, конфиг-таблица впервые получила свои веса (она карточится по *контейнеру* в
  960 px, то есть может сделать это и на 1200-px десктопе), а таблица стилей похудела на 512 байт.
  Проверено на роутере: ячейки списка пакетов поехали с `flex: 0 1 auto` на `1 1 30px` / `2 2 60px` /
  `10 10 300px`.
- **`cssdiff.py` мог обнулить выбор темы на dev-роутере.** Он переключает `luci.main.mediaurlbase` и
  восстанавливает его в `finally`, но читал исходное значение без фолбэка — при сорванном `ssh` оно
  становилось пустой строкой, и восстановление выполняло `uci set luci.main.mediaurlbase=`. У двух
  его соседних тулов фолбэк на bootstrap есть, у этого не было. Теперь он вовсе отказывается менять
  тему, если не смог прочитать значение, нужное чтобы вернуть её назад.
- **`preview.py --layout footstrap-top` снимал скриншоты сломанного UI.** Он направлял
  `mediaurlbase` роутера на `/luci-static/footstrap-top` — путь, который остальной репозиторий
  активно удаляет. Раскладка теперь клиентская: она задаётся в браузере, а варианты — `sidebar` /
  `top`.
- **`install.sh` не чистил module-кеш LuCI**, удаляя только index-кеш — а протухший module-кеш сразу
  после установки пакета, заменяющего JS темы, это единственный случай, когда он реально мешает.
- **Тэггер таблиц данных и его же фильтр мутаций использовали разные селекторы.** Тэггер спрашивал
  `table.table`, тогда как соседний фильтр несёт комментарий, объясняющий, почему так нельзя. Все
  `.table`, которые эмитит сток LuCI, действительно `<table>`, так что *здесь* это ничего не стоило —
  но это везение, а правило покрытия гласит, что стороннее `luci-app-*` рисует то, чего сток не
  рисует никогда. Один селектор.
- **Самообновление работало только на роутерах, где случайно оказался установлен `curl`.** `curl` не
  входит в дефолтный набор пакетов OpenWrt — базовый образ везёт `uclient-fetch` — поэтому на стоковом
  роутере и значок обновления в Appearance, и кнопка Update падали с обманчивым `ERR: cannot reach the
  GitHub release API`. Воспроизведено на dev-роутере убиранием `/usr/bin/curl`. Скрипт теперь
  откатывается на `uclient-fetch`, ровно как уже делал `install.sh`, так что тема по-прежнему зависит
  только от `luci-base`.
- **Установка или обновление темы больше не разлогинивает всех пользователей LuCI.** `postinst` делал
  `/etc/init.d/rpcd restart`, а rpcd держит сессии в памяти. `reload` (SIGHUP) перечитывает
  `/usr/share/rpcd/acl.d/*` — единственное, что этому пакету от rpcd нужно; проверено на живом
  роутере: удаление нашего ACL-файла с последующим `reload` переключает `session access` для скрипта
  самообновления с `true` на `false`, а сессия, созданная до `reload`, его переживает и умирает от
  `restart`. Экран «вы разлогинены, войдите заново», который показывал апдейтер, существовал только
  чтобы объяснить разлогин, который пакет наносил сам себе.
- **Таблица данных, свёрнутая в карточки выше телефонного брейкпоинта, рисовала колонки, которые
  положено прятать, и игнорировала веса колонок.** Стек *измеряется* (`fs-select.js`), поэтому
  срабатывает на любой ширине, но три части одного контракта — собственный `display: block` таблицы,
  колонки `.hide-xs`/`.hide-sm`, которые сток LuCI выкидывает, и веса `.col-N` — жили только внутри
  `@media (max-width: 767px)`. В раскладке с сайдбаром колонка контента равна `viewport − 224 − 56`,
  поэтому примерно между 768 и 860 px она уже ниже порога «слишком тесно для таблицы», а медиазапрос
  уже выключен. Измерено на живом роутере при 790/820/850 px: таблица аренд свернулась в карточки,
  оставшись `display: table` (сохранив intrinsic min-width, ради предотвращения которого `display:
  block` и существует), а список Wi-Fi-клиентов отрисовал **все пять** своих ячеек `.hide-xs`. Теперь
  эти правила ключуются от класса `.fs-stacked` — там, где стек и решается.
- **Будущая палитра красила бы текст success краской *danger*.** `.cbi-tooltip.success` читал
  `--fs-on-danger`. Увидеть это было нечем: все поставляемые палитры дают всем четырём краскам одно и
  то же значение — `cssdiff` показывал ноль расхождений, `audit.py` видел определённую переменную, axe
  мерил правильный контраст. Доказано живьём: если сделать `--fs-on-danger` красным, тултип success
  краснеет, а `--fs-on-good` на него не влияет вообще.
- **`build-css.sh` мог молча испортить CSS-строку.** Финальный проход «убрать последнюю `;` блока» был
  `sed 's/;}/}/g'`, прикрученным к string-aware awk, а sed не видит строк: `content: ";}"` превращался
  в `content: "}"`, а data-URI с `;}` калечился так же. Оба случая воспроизведены. Теперь срез
  происходит внутри сканера, который и так отслеживает кавычки; вывод на текущем дереве байт-в-байт
  прежний, так что в отгрузке ничего не меняется.
- **Установка footstrap могла переписать активную тему роутера, на котором стоит чужая тема.** Проверка
  «а лежит ли активная тема на диске?» в `uci-defaults` выполнялась над той темой, что была текущей, а
  не над нашей — поэтому стороннюю тему, чей каталог ucode-шаблона назван не так, как basename её
  медиа-каталога, могло молча заменить на bootstrap. Теперь проверка ограничена
  `/luci-static/footstrap*`: чиним то, что везём сами, остальной роутер не трогаем.
- **Установщик советовал новым пользователям выбрать записи темы, которых больше нет**
  (`FootstrapSidebar` / `FootstrapOnTop`). Запись одна — `Footstrap`; раскладка переключается в
  Appearance и живёт в браузере.

### Безопасность
- **`install.sh` больше не отключает проверку TLS-сертификата.** Установщик пользователи пайпят из
  интернета в `sh` от рута, а он повторял любую загрузку с `--no-check-certificate` (или `curl -k`)
  после *любого* сбоя проверенной попытки — включая MITM с подложным сертификатом. Скачанное затем
  ставилось как root-пакет. Подсказка «поставьте CA bundle», которую он печатает, была недостижима
  именно в том случае, ради которого написана. `ca-bundle` входит в `DEFAULT_PACKAGES` OpenWrt, так
  что небезопасный путь ничего не покупал на стоковом роутере и молча разоружал проверку на сломанном.
- **Пакет темы теперь сверяется с sha256, который публикует GitHub — и в установщике, и в
  самообновлении.** Оба ставят через `apk add --allow-untrusted`, то есть без подписи пакета, поэтому
  поле `digest` у ассета в release API — единственная проверка целостности, какая вообще есть, и ни
  один из скриптов её не читал. При несовпадении установка теперь отвергается. Хеш идёт по тому же
  TLS-каналу, что и URL, поэтому он не защищает от компрометации `api.github.com`; от чего он
  защищает — это от обрезанной или подменённой загрузки с CDN ассетов, а это другой хост.
- **Хост ассета пиньятся, а схема редиректа — на тех бэкендах, которые это умеют.** URL загрузки
  читается из ответа API и скармливается `apk add` от рута; теперь он обязан быть хостом GitHub. У
  `curl` дополнительно стоят `--proto '=https' --proto-redir '=https'`, поэтому он не пойдёт за
  редиректом на голый `http://` по пути к CDN. **У `uclient-fetch` — бэкенда, который идёт первым и
  который реально стоит на стоковом роутере, — такого флага нет**, так что на этом пути защищают
  allowlist хостов и sha256, а не пин схемы.
- **`install.sh` качает в `mktemp -d`, а не в предсказуемый `/tmp/footstrap-install`.** `/tmp` имеет
  права 1777, поэтому любой локальный непривилегированный процесс мог заранее создать это имя как
  симлинк и заставить рута записать пакет через него в файл по своему выбору (CWE-377) — ровно та
  гонка, о недопущении которой `footstrap-selfupdate.sh` уже пишет шесть строк комментария.
- **Самообновление больше не может запустить две установки разом.** Проверка «идёт ли уже установка?»
  была чтением с последующей записью, поэтому два одновременных RPC оба читали «нет» и оба порождали
  `apk add` над одним пакетом — воспроизведено запуском скрипта дважды сразу. Её заменил атомарный
  lock через `mkdir`, где признаком протухания служит mtime самого lock-каталога (пять одновременных
  вызовов теперь дают ровно один `STARTED` и четыре `RUNNING`).
- **jsmin и i18n-сканер в CI пиньятся по SHA коммита и сверяются по контрольной сумме.** Оба тянулись
  с `openwrt/luci@master` и затем *исполнялись* — jsmin компилируется из C и служит гейтом, который
  решает, безопасен ли отгружаемый JavaScript. С подвижной ветки гейтом становится то, что апстрим
  запушил последним.

## [0.8.0] — 2026-07-13

### Добавлено
- **Раскладка «сайдбар / верхняя панель» стала мгновенным переключателем в Appearance → Layout и
  запоминается в браузере.** Раньше это был *серверный* выбор: две записи темы в System → Design
  (`FootstrapSidebar`, `FootstrapOnTop`), у каждой свой `mediaurlbase`, свой каталог шаблонов и свой
  рендерер меню; смена шла через страницу Design и перезагрузку. Теперь это клиентская настройка, как
  тёмная тема — `:root[data-layout]`, пре-пейнт в `head.ut` до первого кадра, поэтому мигания нет, — и
  переключение перерисовывает страницу на месте **без перезагрузки и без пере-рендера меню**: DOM уже
  обслуживает обе раскладки, а `MutationObserver` меню сворачивает аккордеон в дропдауны (и
  восстанавливает обратно), потому что это ровно то же изменение состояния, что и схлопывание рейки.

- **Три новых CSS-гейта, каждый закрывает дыру, которую не покрывает ничего готового** (`npm run check`
  прогоняет все; все три — только для CI, сборочному боту OpenWrt по-прежнему нужен один `cat`).
  - `tools/css-dup.mjs` — **одинаковое тело правила под двумя разными гардами.** Ни один линтер это не
    пометит и никогда не пометит: для каскад-aware инструмента два правила под взаимоисключающими
    гардами (media-запрос против атрибутного селектора, класс против container-query) **оба
    необходимы**, ведь совпасть может только одно. Но именно эта форма и разъезжается. В этом релизе
    удалено 55 таких деклараций в хроме, а число дублирующихся тел упало с **4 групп (~41 избыточная
    декларация) до 2 (~14)**; остаток инструмент держит под бюджетом — то, что язык CSS действительно
    вынуждает, остаётся видимым и не может расти.
  - `tools/fs-orphans.mjs` — **мёртвый CSS, ограниченный неймспейсом `fs-*`.** PurgeCSS/uncss и
    coverage-обрезка здесь строго опасны: coverage-контракт существует именно потому, что сторонний
    `luci-app-*` рисует виджеты, которых нет ни на одной видимой нам странице, и инструмент, срезающий
    ненаблюдённое, разденет чужое приложение. Но `fs-`-класс не может выпустить никто, кроме нас, — и
    внутри этого одного неймспейса «никто не рендерит» действительно означает «мёртвый», без риска для
    контракта. Это и есть проверка, ловящая селектор, забытый после удаления его разметки.
  - `tools/css-metrics.mjs` — **ratchet** на `!important` (33: 16 в theme/pages, каждый воюет с inline
    или неслоёной декларацией, плюс 17 в base), максимальную специфичность и пустые правила. `stylelint`
    не даст *новому* файлу добавить `!important`; это не даст разрешённым файлам тихо обрасти новыми.

### Изменено
- **В Design теперь ОДНА тема «Footstrap» вместо двух.** `mediaurlbase` всегда
  `/luci-static/footstrap`, раскладка вообще перестала быть серверной темой. Роутер, который был на
  верхней панели, её сохраняет: `uci-defaults` записывает её как дефолт роутера
  (`luci.main.footstrap_layout=top`), а `head.ut` штампует это в `<html data-layout>` — так что
  *мигрированный роутер открывается верхней панелью даже в браузере, который его никогда не видел*, а
  собственный выбор пользователя (localStorage) навсегда перекрывает дефолт. Shell-скрипт не может
  записать localStorage; это и есть канал, переносящий факт через обновление.
- **`data-layout` всегда штампуется явным значением, сервером.** Если бы «отсутствие = сайдбар», все
  правила пришлось бы писать *негативно* (`:not([data-layout="top"])`), и будущая третья раскладка
  молча унаследовала бы правила сайдбара просто потому, что она «не top». Вместо этого каждое правило
  раскладки — позитивное совпадение, и новая раскладка обязана подписаться на него явно. Побочно: хром
  корректен и с отключённым JS — атрибут есть до первого байта скрипта.
- **На телефоне любая раскладка рисует одну и ту же верхнюю панель.** Телефонная панель сайдбара и
  панель верхней раскладки — теперь один вид. Там, где раскладки расходились в стиле одного и того же
  элемента, выжило значение верхней панели.
- **«Выход» в верхней панели — тот же контрол, что и в сайдбаре**: квадратная иконка в правом кластере,
  а не отдельный текстовый пункт меню. Рендерер меню и так выбрасывал узел `admin/logout` в пользу
  собственного контрола темы — верхняя раскладка несла оба.
- **Панель теперь описана один раз, а вертикальный сайдбар стал исключением.** Панель нужна при
  `(ширина ≤ 767px) ИЛИ (:root[data-layout="top"])`, а CSS не умеет объединить media-запрос с атрибутным
  селектором в одном селекторе — поэтому под двумя гардами панель приходилось писать дважды. Замерено:
  **55 из ~75 деклараций были идентичны**, то есть могли молча разъехаться. Инверсия — панель как
  безусловная база, вертикальный сайдбар как единственный гард-оверрайд, выигрывающий по специфичности
  (`0,3,0` против `0,1,0`), а не по порядку в файле — описывает каждое из трёх состояний хрома ровно один
  раз. Это стало возможно только потому, что `data-layout` всегда несёт явное значение. `cssdiff`
  доказывает, что инверсия не изменила ничего, чего не собиралась: **ноль непреднамеренных расхождений
  свойств на 3 014 элементах**, а единственные расхождения — те два, которых релиз и добивался (перенос
  хостнейма и активный пункт mode-меню, теперь тематизированный во всех раскладках, а не только в
  верхней).
- **Панель сбрасывает меню на вторую строку только когда оно не влезает — по измерению, а не по догадке.**
  Раньше это был `@media (max-width: 1199px)`. Но влезет ли меню рядом с брендом, зависит от того, сколько
  секций у ЭТОГО роутера (сток рисует 5, коробка с парой `luci-app-*` — одиннадцать), поэтому брейкпоинт по
  ширине устройства — неверный инструмент. Замерено: содержимое панели у стокового роутера ~683px, то есть
  одна строка держится до ~723px — **тот брейкпоинт сбрасывал меню на каждом ноутбуке, отнимая строку
  впустую**. Теперь меню сначала ужимает пилюли (`.fs-dense1/2`) и стекается, только если и самый плотный
  шаг всё ещё переносится: на 1000px стоковое 5-секционное меню не стекается никогда, 11 секций ужимаются,
  13+ — стекаются.

- **Один движок измерения (`fs-fit.js`) на все решения «а оно ещё влезает?».** В теме два места должны
  решить то, чего не умеет спросить ни один CSS-запрос, потому что ответ зависит от того, сколько нужно
  КОНТЕНТУ, а не от того, насколько широк экран: влезает ли меню рядом с брендом, и читается ли таблица
  ещё как таблица. И то и другое когда-то было брейкпоинтом, и каждое из тех чисел было догадкой,
  которую какой-нибудь реальный роутер опровергал. Форма ответа всегда одна — померить элемент в
  РАЗВЁРНУТОМ виде и переключить класс, — поэтому измерение, склейка по кадрам и ResizeObserver живут в
  одном модуле, а вызывающий приносит только решение. Движок закрепляет три правила, каждое из которых
  — реально пойманный баг: **мерить развёрнутым** — свёрнутое всегда «влезает» (стекнутая таблица это
  стопка flex-строк), поэтому замер как есть её разворачивает, а следующий кадр сворачивает обратно, и
  это осцилляция. Второе — страховка, а не лечение: **пере-мерять синхронно на мутации**, потому что
  колбэк `MutationObserver` — микротаск и выполняется *до* отрисовки кадра, а `requestAnimationFrame` —
  *в момент* отрисовки; и если поллер когда-нибудь ПЕРЕСОЗДАСТ элемент таблицы (свежий приходит без
  нашего класса), отложенный путь отрисует один кадр на полную ширину. На этом роутере — не
  пересоздаёт: LuCI обновляет ячейки на месте, класс переживает тик, и 60 замеров за 6 тиков поллинга
  не показывают моргания ни на одном из путей. Синхронный замер стоит одного layout на пачку мутаций и
  убирает риск заранее. Третье — склеивать замеры при ресайзе.

- **Сайдбар уступает место панели, когда КОЛОНКЕ КОНТЕНТА становится слишком тесно, — поэтому иконочная
  рейка держится примерно на 155px дольше развёрнутого сайдбара.** Раньше на обоих был один
  вьюпортный брейкпоинт (767px), и он не мог быть верным: сайдбар отъедает не константу, а 224px в
  развёрнутом виде и 68px в виде рейки. То есть свернуть сайдбар значило вернуть контенту ~156px — и
  тут же схлопнуть всё целиком на той же самой ширине окна: освобождённое место не давало
  пользователю ничего. Теперь решение считается от реальной доли сайдбара против объявленного минимума
  (500px контента): развёрнутый сайдбар уступает на ~780px, рейка — на ~625px.

- **Индикатор поллинга теперь выглядит как один из контролов, рядом с которыми стоит.** В панели за ним
  идут кнопки Appearance и «Выход», а он был капсулой 28px рядом с двумя скруглёнными квадратами 34px —
  три размера и две формы там, где строка одна. Теперь он берёт высоту кнопок и радиус контролов темы; а
  в вертикальном сайдбаре, где его соседи — не квадратные кнопки, а строки во всю ширину, он тянется на
  всю колонку, а не висит чипом посередине.

### Удалено
- **Второй рендерер меню, второй шаблон и второй стиль удалены** (`menu-footstrap-top.js`,
  `ucode/template/themes/footstrap-top/`, симлинк `/luci-static/footstrap-top`,
  `styles/theme/50-topnav.css`). Это никогда не были два дизайна: рендерер сайдбара уже выдавал
  разметку, которую его же CSS превращает в горизонтальную панель на телефоне, и у него уже был
  «flyout-режим», где секция ведёт себя ровно как дропдаун верхнего меню. Верхняя раскладка — это тот
  же режим на ширине десктопа: **вся уникальная логика удалённого рендерера уместилась в одну функцию**
  (`clampDropdown`, сдвигающая дропдаун обратно в вьюпорт у правого края), и она переехала в
  выживший рендерер. Раскрытие по наведению всегда было чистым CSS. Удаление второго стиля почти в
  точности оплатило новый: **сам мердж раскладок стоил +78 байт CSS**. (Релиз целиком — +971 байт;
  остальное покупает перенос хостнейма, измеряемый стекинг и сайдбар по ширине контента.)

### Исправлено
- **На странице, где нечего поллить (Software, Backup…), висела пилюля «Paused» — отчёт о поллинге,
  которого там нет.** LuCI показывает индикатор по событию `poll-start` и переключает его в «Paused» по
  `poll-stop` — и больше никогда не прячет (`ui.hideIndicator()` существует, но ядро зовёт его только
  для `uci-changes`). При полной загрузке этого не видно: `Poll.start()` шлёт `poll-start` только при
  непустой очереди, поэтому страница без поллеров индикатора просто не заводит. Но SPA-роутер этой темы
  на каждой навигации чистит очередь и зовёт `stop()`, а `stop()` **шлёт** `poll-stop`. Теперь пилюля
  подчиняется единственному осмысленному правилу — она существует тогда и только тогда, когда есть что
  поллить: исчезает на неполлящейся странице и возвращается на поллящейся. **Ручная** пауза
  по-прежнему показывает «Paused», потому что там очередь не пуста и слово что-то значит. Побочно это
  лечит тот же призрак и в стоковом LuCI, где удаление последнего поллера останавливает цикл и
  оставляет пилюлю висеть.
- **Свёрнутый в рейку сайдбар при сужении окна РАСПАХИВАЛСЯ обратно, а повторное переключение уводило
  сразу в мобильную панель.** Правила рейки были под `min-width: 768px`, а гард вертикального сайдбара
  уехал на 521px — и в зазоре между ними вертикальные правила уже действовали, а правила рейки ещё нет:
  сайдбар раскрывался на все 224px, пока окно *уменьшалось*. Рейка — это РЕЖИМ вертикального сайдбара и
  не может существовать там, где нет его самого; теперь у них буквально один и тот же гард.

  крутился.** Мишень, которая не останавливается, не нажимается. Теперь это неподвижный зелёный глиф —
  каким и должен был быть: спиннер обещает, что происходит нечто, *чего вы ждёте*, а тут поллинг тихо
  тикает в фоне вечно, и вечно вращающаяся иконка на периферии зрения просто заставляет простаивающий
  роутер выглядеть занятым. Пауза и снятие паузы кликом работают; а на паузе глиф теперь гаснет, а не
  светит зелёным, вводя в заблуждение.
- **Пилюля «Refreshing» отлипала от кнопок Appearance и «Выход» в панели** вместо того, чтобы
  прижиматься к ним. `margin-left: auto` стоял и у неё, и у кнопок, а два auto ДЕЛЯТ свободное место
  между собой, вместо того чтобы прижать один кластер к правому краю.
- **В свёрнутой рейке вернулись шевроны у пунктов меню.** У пункта рейки нет ни подписи, ни аккордеона —
  его дети вылетают вбок, — поэтому шеврон ничего не сообщает и лишь теснит иконку. Правило рейки имело
  специфичность (0,4,0) и проигрывало вертикальному (0,4,3), которое его включает обратно.
- **Список пакетов (apk Software) перестал сворачиваться в строки и вылезал за секцию.** Это
  `<table class="table" id="packages">` — без класса `.cbi-section-table` вовсе, — а его шапка это
  `.cbi-section-table-titles`, а не `.table-titles`, которую искал разметчик data-таблиц. То есть он не
  подходил **ни под одно** правило и требовал собственного рукописного блока стекинга на пятом
  брейкпоинте. Теперь разметчик принимает обе разметки шапки, и список карточится и раз-карточивается
  как любая другая data-таблица.
- **Data-таблица становится карточками, когда она реально перестаёт влезать, а не когда вьюпорт
  пересёк число.** Раньше это был container-query, и порогов было ТРИ — 568 для обычной таблицы, 780
  для DHCP-аренд (их 8 nowrap mono-колонок держат ~736px пола, поэтому карточиться надо раньше) и 800
  для списка пакетов, — причём последние два несли **собственную копию** правил карточки, потому что CSS
  не умеет делить блок деклараций между двумя порогами `@container`. Оба на самом деле спрашивали
  *переполняется ли?*, а это **факт, который вычисляет браузер**, — поэтому оба исчезли: переполнение
  измеряется, и каждая таблица находит свою ширину сама, включая таблицу из стороннего `luci-app-*`,
  число колонок которой мы бы никогда не угадали. Правила карточки теперь существуют в одном экземпляре.
  Осталось лишь то суждение, которого измерение вынести не может («слишком тесно, чтобы вообще быть
  таблицей», 568) — и оно лежит рядом с измерением, а не в стилях.
- **Длинный хостнейм переносится по строкам, а не обрезается молча.** Было `nowrap` + многоточие, что
  скрывало ровно ту строку, которая говорит, КАКОЙ это роутер. Теперь он ломается по строкам —
  `overflow-wrap: anywhere`, то есть разорвёт и посреди слова, если одно «слово» шире бокса, но только
  когда лучшего переноса нет: обычное имя с точками по-прежнему ломается по точкам. Одного переноса было
  мало: панель — flex-wrap, и вместо того чтобы сжать бренд, flexbox преспокойно уносил *меню*, оставляя
  78-символьный хостнейм в одну строку шириной 609px. Поэтому бренд ещё и **ограничен** по ширине (30ch),
  а панель растёт в высоту, чтобы вместить лишнюю строку.

  что локальный пользователь мог заранее создать их симлинками и заставить root-овые
  `cp`, `chmod`, `curl -o` и `>` писать сквозь них в произвольный файл (CWE-377). `PATH`
  прибит гвоздями, потому что rpcd позволяет вызывающему передать env. Оба `curl`
  получили таймауты, а обрезанный кэш больше не заклинивает кнопку обновления до
  перезагрузки.

### Производительность
- JS снова минифицируется (83 → 35 КБ). `LUCI_MINIFY_JS:=0` был скопирован с CSS-стороны,
  где он оправдан, потому что csstidy калечит `:has()`/`color-mix()`. Но JS `luci.mk`
  минифицирует через jsmin, который и так есть на сборочном боте, а uhttpd отдаёт `/www`
  без сжатия, так что это были байты и в эфире, и во флеше. Комментарии остаются в git.
  Опасность jsmin реальна и молчалива: он отличает регулярку от деления по одному
  предыдущему символу и может проглотить остаток файла, выйдя с кодом 0. Поэтому
  `wrap-regex` запрещает опасную форму, а `tools/jsmin-verify.mjs` доказывает, что поток
  токенов на выходе совпадает с исходником.
- `build-css.sh` выжимает пробелы, которые CSS игнорирует (117.5 → 108.3 КБ);
  нейтральность доказана `cssdiff` по ~4000 элементов.

## [0.2.x] — 2026-07-08

### Добавлено
- Popover Appearance (Mode auto/light/dark и Palette) вместо простого тумблера тёмной
  темы. Клиентский, мгновенный, запоминается, без перезагрузки.

### Исправлено
- Сохранение настроек Wi-Fi, как и любой формы с нативным select, молча не работало.
  `fs-select` рисовал свой стилизованный дропдаун *перед* нативным `<select>`, поэтому
  дропдаун становился `frameEl.firstChild`, а `ui.Select.getValue()` читает
  `this.node.firstChild.value`: он получал `<div>` и возвращал undefined. Теперь дропдаун
  идёт после select, значение зеркалится в обе стороны.
- Неактивные панели вкладок показывали фантомную полосу ~38 px при первой отрисовке. LuCI
  ставит `data-tab-active="true"` только показанной вкладке, у остальных атрибута ещё нет,
  поэтому правило по значению `"false"` их не ловило.
- MutationObserver, скрывающий дубли, ставился заново на каждый опрос, тёк наблюдателями и
  прогрессивно замедлял страницу.
- Токен `--faint` (заголовки таблиц, подписи полей) использовался, но нигде не объявлялся.
- Кнопка Enabled на System → Startup была синим текстом по зелёной заливке.

## [0.1.x] — 2026-07-08

### Добавлено
- Первый релиз. `luci-theme-footstrap` для OpenWrt 25.12+: две раскладки, серверная
  оболочка целиком на ucode и сборка apk через OpenWrt SDK.

### Исправлено
- Минификаторы CSS и JS в LuCI отключены. csstidy калечит `:has()`, `color-mix()` и
  вложенный `calc()`, что ломало вёрстку напрочь. Минификация JS вернулась в 0.7.12, когда
  безопасность jsmin подтвердил гейт эквивалентности токенов.
